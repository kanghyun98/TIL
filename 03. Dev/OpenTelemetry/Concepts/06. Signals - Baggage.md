# Baggage

- OpenTelemetry에서 Baggage는 context 옆에 존재하는 contextual information입니다. 
- Baggage는 key-value store입니다. 즉, context와 함께 원하는 데이터를 전파할 수 있게 해줍니다.

- Baggage를 사용하면 서비스와 프로세스 사이에서 데이터를 전달할 수 있습니다. 이렇게 전달된 데이터는 해당 서비스의 traces, metrics, logs에 추가할 수 있습니다.  

## **Example**

- Baggage는 tracing에서 추가 데이터를 서비스 간에 전파하기 위해 자주 사용됩니다.

- 예를 들어 요청 시작 시점에 `clientId`가 있다고 가정해보겠습니다.

  - 그런데 이 ID를 다음 위치에서도 사용할 수 있게 만들고 싶을 수 있습니다.

    - trace 안의 모든 span

    - 다른 service의 일부 metrics

    - 처리 과정 중 일부 logs

  - Trace는 여러 service에 걸쳐 있을 수 있기 때문에, 코드베이스 여러 곳에 `clientId`를 복사하지 않고도 이 데이터를 전파할 방법이 필요합니다.

  - Context Propagation을 사용해 이 service들 사이에 baggage를 전달하면, `clientId`를 추가적인 span, metric, log에 추가할 수 있습니다.

- 또한 instrumentation은 baggage를 자동으로 전파해줍니다.

## What should OTel Baggage be used for?

- Baggage는 보통 요청 시작 시점에만 사용할 수 있는 정보를 downstream에서도 사용할 수 있게 포함하는 데 가장 적합합니다.

  - Account Identification, User IDs, Product IDs, origin IPs

  - 이 정보를 baggage를 사용해 전파하면 backend에서 telemetry를 더 깊게 분석할 수 있습니다.

  - 또한 downstream operation에 대한 정보를 log로 남기면서, 동일한 User ID를 log data에 포함할 수도 있습니다.

## Baggage security considerations

- 민감한 Baggage item은 third-party API 같은 의도하지 않은 resource와 공유될 수 있습니다.

  - 그 이유는 automatic instrumentation이 service의 대부분의 network request에 Baggage를 포함하기 때문입니다.

  - 구체적으로 Baggage와 trace context의 다른 부분들은 HTTP header로 전송됩니다. 따라서 network traffic을 검사하는 사람에게 노출될 수 있습니다.

- Traffic이 네트워크 내부로 제한되어 있다면 이 위험이 적용되지 않을 수도 있습니다. 하지만 downstream service가 Baggage를 네트워크 외부로 전파할 수 있다는 점은 염두에 두어야 합니다.
- 또한 Baggage item이 실제로 자신이 만든 것인지 보장하는 built-in integrity check는 없습니다. 따라서 Baggage를 읽을 때는 주의해야 합니다.  

<br>

## Baggage is not the same as attributes

- Baggage에 대해 중요한 점은, baggage가 별도의 key-value store라는 것입니다.
- 명시적으로 추가하지 않는 한, baggage는 span, metric, log의 attributes와 연결되지 않습니다.

- Baggage entry를 attributes에 추가하려면, baggage에서 데이터를 명시적으로 읽은 뒤 span, metric, log의 attributes로 추가해야 합니다.

- Baggage의 일반적인 사용 사례 중 하나는 전체 trace에 걸쳐 데이터를 Span Attributes에 추가하는 것입니다.
- 그래서 여러 언어에는 span 생성 시점에 baggage의 데이터를 span attribute로 추가하는 Baggage Span Processor가 있습니다.

# Framework Routers

## 1️⃣ **HydratedRouter**

> [https://reactrouter.com/api/framework-routers/HydratedRouter](https://reactrouter.com/api/framework-routers/HydratedRouter)
> 
- `HydratedRouter`는 Framework Mode에서 사용하는 라우터 컴포넌트이다.
- `ServerRouter`에서 생성된 라우터를 hydration 하기 위해 사용된다.

### ▶️ Props

- **`getContext`**
    - `createBrowserRouter`에 전달될 context factory 함수이다.
    - 이 함수는 각 navigation 또는 fetch 마다 새로운 context 인스턴스를 생성하기 위해 호출된다.
    - 생성된 context는 `clientAction` 또는 `clientLoader` 함수에서 사용할 수 있다.
- **`onError`**
    - 애플리케이션에서 발생하는 middleware, loader, action 또는 render 오류를 처리하기 위한 에러 핸들러 함수이며, 오류가 발생할 때 호출된다.
    - `ErrorBoundary`와 달리 리렌더링의 영향을 받지 않으며 오류마다 한 번만 실행된다.
    - 로깅 또는 오류 보고에 사용할 수 있다.

## 2️⃣ Server Router

> [https://reactrouter.com/api/framework-routers/ServerRouter](https://reactrouter.com/api/framework-routers/ServerRouter)
> 
- ServerRouter는 Framework Mode에서 사용하는 React Router 애플리케이션의 서버 엔트리 포인트이다.
- 이 컴포넌트는 서버 응답에서 HTML을 생성하는 데 사용된다.

### ▶️ Props

- `context`
    - 렌더링에 필요한 manifest, route modules 및 기타 데이터가 포함된 entry context이다.
- `nonce`
    - Content Security Policy(CSP) 준수를 위한 선택적 nonce 값이다.
    - 인라인 스크립트를 안전하게 실행할 수 있도록 허용하는 데 사용된다.
- `url`
    - 처리 중인 요청(request)의 URL이다.

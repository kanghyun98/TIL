# Hooks API

## 1️⃣ Navigation Hooks

### ▶️ `useNavigate`

- 브라우저에서 **프로그래밍 방식으로 네비게이션**할 수 있는 함수를 반환한다.
- 사용자 상호작용이 아닌 상황에서도 경로 이동이 필요한 경우 사용할 수 있다 (예: inactivity 후 로그아웃, 타이머 UI 등)
- 반환되는 함수 시그니처는 `navigate(to, options?)` 또는 `navigate(delta)` 형태
    - `to`: 문자열 경로 또는 `To` 객체
    - `delta`: 숫자 (히스토리 스택 앞/뒤 이동)
- https://reactrouter.com/api/hooks/useNavigate

```tsx
import { useNavigate } from "react-router";

function SomeComponent() {
  let navigate = useNavigate();
  return (
    <button onClick={() => navigate(-1)}>
      Go Back
    </button>
  );
}
```

### ▶️ `useNavigation`

- 현재 네비게이션 상태를 반환한다.
- 네비게이션이 진행 중이면 해당 상태를 나타내고, 아니라면 `"idle"` 상태를 반환한다.
- https://reactrouter.com/api/hooks/useNavigation

### ▶️ `useNavigationType`

- 현재 location에 도달한 navigation action을 반환한다.
- 반환값: `"POP"`, `"PUSH"`, `"REPLACE"`

## 2️⃣ Location & URL Hooks

### ▶️ `useLocation`

- 현재 [Location 객체](https://api.reactrouter.com/v7/interfaces/react-router.Location.html)를 반환한다.
- `pathname`, `search`, `hash`, `state`, `key`를 포함한다.

```tsx
const location = useLocation()

interface Location<State = any> {
    hash: string;
    key: string;
    pathname: string;
    search: string;
    state: State;
    unstable_mask: undefined | Path;
}
```

### ▶️ `useSearchParams`

- 현재 URL의 [`URLSearchParams`](https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams)와 이를 업데이트하는 함수를 반환한다.
- `setSearchParams` 함수
    - 검색 파라미터를 설정하면 새로운 URL로 이동된다.

```tsx
const [searchParams, setSearchParams] = useSearchParams();
```

### ▶️ `useHref`

- `to` 값을 현재 location 기준으로 resolve한 `href` 문자열을 반환한다.

```tsx
const href = useHref("some/where");
```

### ▶️ `useResolvedPath`

- `to`를 현재 location 기준으로 resolve한 `Path` 객체를 반환한다.

```tsx
// if the user is at /dashboard/profile
const path = useResolvedPath("../accounts");
path.pathname; // "/dashboard/accounts"
path.search; // ""
path.hash; // ""
```

## 3️⃣ Route Matching Hooks

### ▶️ `useParams`

- 현재 URL에서 매칭된 route params를 반환한다.

```tsx
// 라우터 패턴이 /posts/:postId 인 경우
// /posts/123
const params = useParams();
console.log(params.postId); // 123
```

### ▶️ `useMatch`

- 주어진 패턴이 현재 URL과 매칭되면 [`PathMatch`](https://api.reactrouter.com/v7/interfaces/react-router.PathMatch.html) 객체를 반환한다.
- 매칭되지 않으면 `null`을 반환한다.
- `active` 상태인 컴포넌트를 알아야 할 때 용이함

### ▶️ `useMatches`

- 현재 매칭된 모든 route 정보를 배열로 반환한다.
- 각 route의 data, params 등에 접근할 수 있다.

### ▶️ `useRoutes`

- route 객체 배열을 받아 라우트 트리를 렌더링한다.
- `<Routes>` 대신 훅을 사용해 **동적인 라우트 렌더링**이 가능하다.
- 반환값은 현재 location에 매칭되는 React element이거나 null이다.

```tsx
const element = useRoutes(routes);
```

### ▶️ `useOutlet`

- 현재 매칭된 자식 라우트의 **React element**를 반환한다.
- `<Outlet>` 컴포넌트가 렌더링하는 자식 라우트의 요소를 얻고자 할 때 사용한다.
- 매칭된 자식 라우트가 없으면 null을 반환한다.

### ▶️ `useOutletContext`

- 가장 가까운 부모 `<Outlet>`이 제공한 **context 값을 반환**한다.
- `<Outlet>`에서 context를 전달하고, 그 값을 자식 라우트 컴포넌트가 이 훅으로 받아서 사용할 수 있다.

```tsx
// Parent route
function Parent() {
  const [count, setCount] = React.useState(0);
  return <Outlet context={[count, setCount]} />;
}
```

```tsx
// Child route
import { useOutletContext } from "react-router";

function Child() {
  const [count, setCount] = useOutletContext();
  const increment = () => setCount((c) => c + 1);
  return <button onClick={increment}>{count}</button>;
}
```

### ▶️ `useInRouterContext`

- 현재 컴포넌트가 <Router>의 하위에 있는지 여부를 판단하여 반환

## 4️⃣ Data Hooks

### ▶️ `useLoaderData`

- 가장 가까운 route의 `loader` 가 반환한 데이터를 반환한다.

```tsx
import { useLoaderData } from "react-router";

export async function loader() {
  return await fakeDb.invoices.findAll();
}

export default function Invoices() {
  let invoices = useLoaderData<typeof loader>();
  // ...
}
```

### ▶️ `useRouteLoaderData`

- 특정 route ID와 연결된 `loader`가 반환한 데이터를 반환한다.

| **Route Filename** | **Route ID** |
| --- | --- |
| `app/root.tsx` | `"root"` |
| `app/routes/teams.tsx` | `"routes/teams"` |
| `app/whatever/teams.$id.tsx` | `"whatever/teams.$id"` |

```tsx
import { useRouteLoaderData } from "react-router";

function SomeComponent() {
  const { user } = useRouteLoaderData("root");
}

// You can also specify your own route ID's manually in your routes.ts file:
route("/", "containers/app.tsx", { id: "app" })
useRouteLoaderData("app");
```

### ▶️ `useActionData`

- 가장 최근에 실행된 `action`이 반환한 데이터를 반환한다.
- 폼 제출이나 fetcher 같은 action 호출 후 그 결과에 접근하는 데 사용한다.

```tsx
import { Form, useActionData } from "react-router";

export async function action({ request }) {
  const body = await request.formData();
  const name = body.get("visitorsName");
  return { message: `Hello, ${name}` };
}

export default function Invoices() {
  const data = useActionData();
  return (
    <Form method="post">
      <input type="text" name="visitorsName" />
      {data ? data.message : "Waiting..."}
    </Form>
  );
}
```

### ▶️ `useRevalidator`

- 데이터 재검증(revalidation)을 트리거하거나 상태를 확인할 수 있는 도구를 제공한다.
- revalidation은 서버 또는 클라이언트에서 데이터를 다시 불러오도록 하는 과정이다.

```tsx
import { useRevalidator } from "react-router";

function WindowFocusRevalidator() {
  const revalidator = useRevalidator();

  useFakeWindowFocus(() => {
    revalidator.revalidate();
  });

  return (
    <div hidden={revalidator.state === "idle"}>
      Revalidating...
    </div>
  );
}
```

## 5️⃣ Submission & Fetcher Hooks

### ▶️ `useSubmit`

- `<Form>`의 명령형 버전
- 코드로 form submit을 트리거할 수 있다.

```tsx
import { useSubmit } from "react-router";

function SomeComponent() {
  const submit = useSubmit();
  return (
    <Form onChange={(event) => submit(event.currentTarget)} />
  );
}
```

### ▶️ `useFetcher`

- navigation 없이 `loader`, `action`과 상호작용할 수 있는 `fetcher` 객체를 반환한다.
- fetcher는 내부적으로 상태 가져오기, 제출, 응답 처리를 관리한다.

```tsx
import { useFetcher } from "react-router"

function SomeComponent() {
  let fetcher = useFetcher()

  // states are available on the fetcher
  fetcher.state // "idle" | "loading" | "submitting"
  fetcher.data // the data returned from the action or loader

  // render a form
  <fetcher.Form method="post" />

  // load data
  fetcher.load("/some/route")

  // submit data
  fetcher.submit(someFormRef, { method: "post" })
  fetcher.submit(someData, {
    method: "post",
    encType: "application/json"
  })

  // reset fetcher
  fetcher.reset()
}
```

### ▶️ `useFetchers`

- 현재 진행 중인 모든 fetcher 배열을 반환한다.

## 6️⃣ Blocking & Browser Hooks

### ▶️ `useBlocker`

- SPA 네비게이션을 차단할 수 있다.
- 사용자가 페이지를 떠날 때나 라우트를 변경하려 할 때, 이를 중단시키거나 확인 UI를 보여줄 수 있다
- 하드 리로드는 차단하지 않는다.

```tsx
// Boolean version
const blocker = useBlocker(value !== "");

// Function version
const blocker = useBlocker(
  ({ currentLocation, nextLocation, historyAction }) =>
    value !== "" &&
    currentLocation.pathname !== nextLocation.pathname
);
```

- **`state`**
    - `unblocked` - blocker가 idle 상태이며 어떤 네비게이션도 막지 않은 상태
    - `blocked` - blocker가 네비게이션을 막은 상태
    - `proceeding` - blocker가 차단된 네비게이션을 진행 중인 상태
- **`location`**
    - blocked 상태일 때, 차단된 네비게이션의 대상 Location을 나타낸다.
    - proceeding 상태일 때, `blocker.proceed()` 호출 이후 이동 중인 location을 나타낸다.
- **`proceed()`**
    - blocked 상태일 때 `blocker.proceed()`를 호출하면, 차단되었던 location으로 네비게이션을 진행한다.
- **`reset()`**
    - blocked 상태일 때 `blocker.reset()`을 호출하면 blocker를 unblocked 상태로 되돌리고, 사용자는 현재 location에 그대로 머무른다.

```tsx
import { useCallback, useState } from "react";
import { BlockerFunction, useBlocker } from "react-router";

export function ImportantForm() {
  const [value, setValue] = useState("");

  const shouldBlock = useCallback<BlockerFunction>(
    () => value !== "",
    [value]
  );
  const blocker = useBlocker(shouldBlock);

  return (
    <form
      onSubmit={(e) => {
        e.preventDefault();
        setValue("");
        if (blocker.state === "blocked") {
          blocker.proceed();
        }
      }}
    >
      <input
        name="data"
        value={value}
        onChange={(e) => setValue(e.target.value)}
      />

      <button type="submit">Save</button>

      {blocker.state === "blocked" ? (
        <>
          <p style={{ color: "red" }}>
            Blocked the last navigation to
          </p>
          <button
            type="button"
            onClick={() => blocker.proceed()}
          >
            Let me through
          </button>
          <button
            type="button"
            onClick={() => blocker.reset()}
          >
            Keep me here
          </button>
        </>
      ) : blocker.state === "proceeding" ? (
        <p style={{ color: "orange" }}>
          Proceeding through blocked navigation
        </p>
      ) : (
        <p style={{ color: "green" }}>
          Blocker is currently unblocked
        </p>
      )}
    </form>
  );
}
```

### ▶️ `usePrompt`

> experimental
> 
- `useBlocker`로 커스텀 UI를 만드는 대신, 사용자에게 `window.confirm` 프롬프트를 보여주기 위한 `useBlocker`의 래퍼

### ▶️ `useBeforeUnload`

- 브라우저 `beforeunload` 이벤트에 콜백을 등록한다.

## 7️⃣ Async & View Transition Hooks

### ▶️ `useAsyncValue`

- 가장 가까운 `<Await>`에서 resolve된 값을 반환한다.

```tsx
function SomeDescendant() {
  const value = useAsyncValue();
  // ...
}

// somewhere in your app
<Await resolve={somePromise}>
  <SomeDescendant />
</Await>;
```

### ▶️ `useAsyncError`

- 가장 가까운 `<Await>`에서 reject된 값을 반환한다.

```tsx
import { Await, useAsyncError } from "react-router";

function ErrorElement() {
  const error = useAsyncError();
  return (
    <p>Uh Oh, something went wrong! {error.message}</p>
  );
}

// somewhere in your app
<Await
  resolve={promiseThatRejects}
  errorElement={<ErrorElement />}
/>;
```

### ▶️ `useViewTransitionState`

- 지정한 location으로의 View Transition 활성 상태를 반환한다.

### ▶️ `useLinkClickHandler`

- `<Link>` 클릭 동작을 처리한다.
- 커스텀 Link 컴포넌트를 만들 때 사용한다.

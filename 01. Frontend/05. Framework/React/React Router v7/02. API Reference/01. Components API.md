# Components API

## 1️⃣ Routing Structure

### ▶️ `<Routes>`

- 현재 location에 가장 잘 일치하는 `<Route>`들을 찾아 해당 UI 브랜치를 렌더링한다.
- 자식으로 `<Route>` 요소들을 가진다.

```tsx
<Routes>
  <Route path="/" element={<Home />} />
</Routes>
```

### ▶️ `<Route>`

- URL 패턴이 현재 location과 일치할 때 특정 요소를 렌더링한다.
- 반드시 `<Routes>` 내부에서 사용해야 한다.

주요 props:

- `path`, `element`, `index`, `errorElement`

### ▶️ `<Outlet>`

- 중첩된 자식 라우트를 렌더링하는 자리 표시자다.
- 부모 라우트 내부에서 자식 라우트를 표시할 위치를 지정한다.

```tsx
import { Outlet } from "react-router";

export default function SomeParent() {
  return (
    <div>
      <h1>Parent Content</h1>
      <Outlet />
    </div>
  );
}
```

### ▶️ `<Navigate>`

- 렌더링 시 즉시 다른 경로로 이동시킨다.
- `to`, `replace`, `state` 등의 props를 지원한다.

```tsx
<Navigateto="/login"replace/>
```

## 2️⃣ Navigation

### ▶️ `<Link>`

- `<Link>`는 **클라이언트 사이드 라우팅을 활성화하는 점진 향상된 `<a href>` 래퍼(wrapper)**
- 클릭 시 전체 페이지 리로드 없이 라우터가 지정한 경로로 네비게이션할 수 있게 한다.

주요 props:

- `to`, `replace`, `state`, `relative`

```tsx
import { Link } from "react-router";

<Link to="/dashboard">Dashboard</Link>;

<Link
  to={{
    pathname: "/some/path",
    search: "?query=string",
    hash: "#hash",
  }}
/>;
```

### ▶️ `<NavLink>`

- `<Link>`와 동일하지만 활성(active) 상태와 대기(pending) 상태에 따라 추가적인 속성을 제공한다.
- `isActive`, `isPending`, `isTransitioning` 상태를 제공한다.
- `className`, `style`, `children`을 함수로 받아 상태 기반 렌더링이 가능하다.

```tsx
<NavLink
  to="/messages"
  className={({ isActive, isPending }) =>
    isPending ? "pending" : isActive ? "active" : ""
  }
>
  Messages
</NavLink>
```

## 3️⃣ Data & Forms

### ▶️ `<Form>`

- `<Form>`은 progressively enhanced HTML `<form>` 컴포넌트
- `<Form>`은 **fetch를 통해 action으로 데이터를 제출**한다
- 제출 시 React Router는 **useNavigation의 pending 상태를 활성화**하여 보다 진보된 사용자 인터페이스를 만들 수 있게 한다
- form의 `action`이 완료되면, React Router는 **페이지의 모든 데이터(loader data)를 자동으로 재검증(revalidate)** 해서 UI가 데이터와 동기화된 상태를 유지한다

```tsx
<Form method="post" action="/projects/1">
  <input name="title" />
</Form>
```

### ▶️ `<Await>`

- Promise 값을 **자동 에러 처리와 함께 렌더링**하는 데 사용된다.
- `<Await>`는 반드시 **`<React.Suspense>` 안에서 렌더링**되어야 한다.

```tsx
import { Await, useLoaderData } from "react-router";

export async function loader() {
  // not awaited
  const reviews = getReviews();
  // awaited (blocks the transition)
  const book = await fetch("/api/book").then((res) => res.json());
  return { book, reviews };
}

function Book() {
  const { book, reviews } = useLoaderData();
  return (
    <div>
      <h1>{book.title}</h1>
      <p>{book.description}</p>
      <React.Suspense fallback={<ReviewsSkeleton />}>
        <Await
          resolve={reviews}
          errorElement={
            <div>Could not load reviews 😬</div>
          }
          children={(resolvedReviews) => (
            <Reviews items={resolvedReviews} />
          )}
        />
      </React.Suspense>
    </div>
  );
}
```

## 4️⃣ Document & Head Management

### ▶️ `<Meta>`

- route module의 `meta` 결과를 `<meta>` 태그로 렌더링한다.
- `<head>` 내부에 위치
- https://reactrouter.com/api/components/Meta

```tsx
import { Meta } from "react-router";

export default function Root() {
  return (
    <html>
      <head>
        <Meta />
      </head>
    </html>
  );
}
```

### ▶️ `<Links>`

- route module의 `links()` 결과를 `<link>` 태그로 렌더링한다.
- `<head>` 내부에 위치
- https://reactrouter.com/api/components/Links

```tsx
import { Links } from "react-router";

export default function Root() {
  return (
    <html>
      <head>
        <Links />
      </head>
      <body></body>
    </html>
  );
}
```

### ▶️ `<Scripts>`

- 애플리케이션의 클라이언트 런타임 스크립트를 렌더링
- `<body>` 내부에 위치
- 서버 렌더링을 사용하는 경우, `<Scripts/>`를 생략해도 앱은 **전통적인 웹 앱처럼 JavaScript 없이** HTML과 브라우저 기능만으로 동작한다.
- https://reactrouter.com/api/components/Scripts

```tsx
import { Scripts } from "react-router";

export default function Root() {
  return (
    <html>
      <head />
      <body>
        <Scripts />
      </body>
    </html>
  );
}
```

### ▶️ `<ScrollRestoration>`

- 브라우저의 기본 스크롤 복원 동작을 에뮬레이션한다.
- 네비게이션 시 스크롤 위치를 복원한다.

```tsx
import { ScrollRestoration } from "react-router";

export default function Root() {
  return (
    <html>
      <body>
        <ScrollRestoration />
        <Scripts />
      </body>
    </html>
  );
}
```

# Rendering Strategies

React Router의 세 가지 렌더링 전략

- Client Side Rendering
- Server Side Rendering
- Static Pre-rendering

### ▶️ **Client Side Rendering**

- 라우트는 사용자가 앱을 탐색할 때 항상 **클라이언트 측에서 렌더링**된다.
- SPA를 만들고 싶다면 서버 렌더링을 비활성화하면 된다.

```tsx
// react-router.config.ts
import type { Config } from "@react-router/dev/config";

export default {
  ssr: false,
} satisfies Config;
```

### ▶️ **Server Side Rendering**

- SSR은 **서버 측에서 HTML을 생성**할 수 있는 배포 환경이 필요하다.
- 전역 설정이지만, **개별 라우트는 정적으로 사전 렌더링될 수 있다.**
- 라우트는 `clientLoader`를 사용해 서버 렌더링/데이터 fetching을 피할 수 있다.

```tsx
// react-router.config.ts
import type { Config } from "@react-router/dev/config";

export default {
  ssr: true,
} satisfies Config;
```

### ▶️ **Static Pre-rendering**

- Pre-rendering은 **빌드 시점에 정적 HTML과 클라이언트 네비게이션 데이터**를 생성
- SEO 및 성능 향상에 유용하며, 서버 렌더링이 없는 배포에서 특히 효과적이다
- 사전 렌더링 시, 라우트 모듈의 loader가 빌드 시점 데이터 fetching에 사용된다.

```tsx
// react-router.config.ts
import type { Config } from "@react-router/dev/config";

export default {
  // build 시점에 prerender할 URL 목록 반환
  async prerender() {
    return ["/", "/about", "/contact"];
  },
} satisfies Config;
```

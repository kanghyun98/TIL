# Routing

### ▶️ Configuring Routes

라우트는 **`app/routes.ts`** 파일에서 구성되며, 두 가지 필수 요소를 가짐

1. URL과 매칭할 **패턴**
2. 해당 라우트의 동작을 정의하는 **Route Module 파일 경로**

```tsx
import {
  type RouteConfig,
  route,
} from "@react-router/dev/routes";

export default [
  route("some/path", "./some/file.tsx"),
  // pattern ^           ^ module file
] satisfies RouteConfig;

// larger sample
export default [
  index("./home.tsx"),
  route("about", "./about.tsx"),

  layout("./auth/layout.tsx", [
    route("login", "./auth/login.tsx"),
    route("register", "./auth/register.tsx"),
  ]),

  ...prefix("concerts", [
    index("./concerts/home.tsx"),
    route(":city", "./concerts/city.tsx"),
    route("trending", "./concerts/trending.tsx"),
  ]),
] satisfies RouteConfig;
```

[**파일 시스템 기반 라우팅](https://reactrouter.com/how-to/file-route-conventions) 방식**을 선호하면, 패키지 `@react-router/fs-routes`가 해당 기능을 제공

```tsx
import {
  type RouteConfig,
  route,
} from "@react-router/dev/routes";
import { flatRoutes } from "@react-router/fs-routes";

export default [
  route("/", "./home.tsx"),

  ...(await flatRoutes()),
] satisfies RouteConfig;
```

### ▶️ Route Modules

`routes.ts`에서 참조된 파일들은 각 라우트의 동작을 정의

```tsx
// Route Module 이란?
route("teams/:teamId", "./team.tsx"),
//           route module ^^^^^^^^
```

```tsx
// sample
// provides type safety/inference
import type { Route } from "./+types/team";

// provides `loaderData` to the component
export async function loader({ params }: Route.LoaderArgs) {
  let team = await fetchTeam(params.teamId);
  return { name: team.name };
}

// renders after the loader is done
export default function Component({
  loaderData,
}: Route.ComponentProps) {
  return <h1>{loaderData.name}</h1>;
}
```

### ▶️ Nexted Routes

중첩 라우트 정의 방식

- `/dashboard`, `/dashboard/settings`

```tsx
import {
  type RouteConfig,
  route,
  index,
} from "@react-router/dev/routes";

export default [
  // parent route
  route("dashboard", "./dashboard.tsx", [
    // child routes
    index("./home.tsx"),
    route("settings", "./settings.tsx"),
  ]),
] satisfies RouteConfig;
```

부모 라우트는 자식 라우트를 렌더링하기 위해 `<Outlet />`을 사용, `<Outlet />`은 현재 매칭된 자식 라우트를 해당 위치에 렌더링한다.

```tsx
import { Outlet } from "react-router";

export default function Dashboard() {
  return (
    <div>
      <h1>Dashboard</h1>
      {/* will either be home.tsx or settings.tsx */}
      <Outlet />
    </div>
  );
}
```

### ▶️ Root Route

`routes.ts`에 정의된 모든 라우트는 `app/root.tsx` 모듈 내부에 중첩됨

### ▶️ Layout Routes

레이아웃 라우트는 URL 경로를 추가하지 않으면서 공통 레이아웃을 공유할 수 있게 해줌

```tsx
// layout.tsx는 /about과 /contact에 공통으로 적용
export default [
  layout("./marketing/layout.tsx", [
    index("./marketing/home.tsx"),
    route("contact", "./marketing/contact.tsx"),
  ])
] satisfies RouteConfig;
```

### ▶️ Index Routes

Index 라우트는 부모의 URL에서 부모의 `<Outlet>` 안에 렌더링된다. (like a default child route)

```tsx
import {
  type RouteConfig,
  route,
  index,
} from "@react-router/dev/routes";

export default [
  // renders into the root.tsx Outlet at /
  index("./home.tsx"),
  route("dashboard", "./dashboard.tsx", [
    // renders into the dashboard.tsx Outlet at /dashboard
    index("./dashboard-home.tsx"),
    route("settings", "./dashboard-settings.tsx"),
  ]),
] satisfies RouteConfig;
```

### ▶️ Route Prefixes

`prefix` 는 **부모 라우트 추가 없이** 하위 라우트들의 경로 앞에 일괄적으로 접두사(path segment)를 붙여주는 helper 함수

```tsx
// This usage of `prefix`...
prefix("parent", [
  route("child1", "./child1.tsx"),
  route("child2", "./child2.tsx"),
])

// ...is equivalent to this:
[
  route("parent/child1", "./child1.tsx"),
  route("parent/child2", "./child2.tsx"),
]
```

### ▶️ **Dynamic Segments**

- 경로 세그먼트가 `:`로 시작하면 dynamic segment
- 라우트가 URL과 매칭되면 dynamic segment 값이 URL에서 파싱된다.
- 파싱된 값은 다른 라우터 API에 `params`로 제공된다.

```tsx
// app/routes.ts
route("teams/:teamId", "./team.tsx"),
```

```tsx
// app/team.tsx
import type { Route } from "./+types/team";

export async function loader({ params }: Route.LoaderArgs) {
  //                           ^? { teamId: string }
}

export default function Component({
  params,
}: Route.ComponentProps) {
  params.teamId;
  //        ^ string
}
```

### ▶️ Optional Segments

- 경로 세그먼트 끝에 `?`가 붙으면 optional segment

```tsx
route(":lang?/categories", "./categories.tsx"),

// optional static segments
route("users/:userId/edit?", "./user.tsx");
```

### ▶️ Splats

- 경로에서 `*`는 splat이라고 함 (aka `catchall`, `star` segments)
- 라우트 경로가 `/*` 으로 끝나면, 뒤에 오는 모든 경로와 매칭된다.
- splat을 사용하면 어떤 라우트와도 매칭되지 않는 요청을 처리할 수도 있다 (404)

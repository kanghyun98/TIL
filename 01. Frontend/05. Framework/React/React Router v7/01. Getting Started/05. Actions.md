# Actions

- 데이터 변경(Data mutations)은 Route action을 통해 수행된다.
- action이 완료되면, 페이지의 모든 loader 데이터가 재검증되어 별도의 코드를 작성하지 않아도 UI가 데이터와 동기화된다
- `action`으로 정의된 Route action은 서버에서만 호출된다.
- `clientAction`으로 정의된 Route action은 브라우저에서 실행된다.

### ▶️ **Client Actions**

- Client Action은 **브라우저에서만 실행된다**
- Server Action과 둘 다 정의되어 있을 때는 Client Action이 우선 실행된다.

```tsx
// route('/projects/:projectId', './project.tsx')
import type { Route } from "./+types/project";
import { Form } from "react-router";
import { someApi } from "./api";

export async function clientAction({
  request,
}: Route.ClientActionArgs) {
  let formData = await request.formData();
  let title = formData.get("title");
  let project = await someApi.updateProject({ title });
  return project;
}

export default function Project({
  actionData,
}: Route.ComponentProps) {
  return (
    <div>
      <h1>Project</h1>
      <Form method="post">
        <input type="text" name="title" />
        <button type="submit">Submit</button>
      </Form>
      {actionData ? (
        <p>{actionData.title} updated</p>
      ) : null}
    </div>
  );
}
```

### ▶️ Server Actions

- Server action은 **서버에서만 실행**되며 클라이언트 번들에서는 제거된다.

```tsx
// route('/projects/:projectId', './project.tsx')
import type { Route } from "./+types/project";
import { Form } from "react-router";
import { fakeDb } from "../db";

export async function action({
  request,
}: Route.ActionArgs) {
  let formData = await request.formData();
  let title = formData.get("title");
  let project = await fakeDb.updateProject({ title });
  return project;
}
```

### ▶️ **Calling Actions**

- Action은 `<Form>`을 통해 선언적으로 호출되며, `useSubmit`(또는 `<fetcher.Form>`과 `fetcher.submit`)을 통해 명령형으로 호출된다
- 라우트의 경로와 `post` 메서드를 참조하여 호출된다

**Calling actions with a Form**

- 내비게이션을 발생시키며 browser history에 새 항목을 추가한다.

```tsx
import { Form } from "react-router";

function SomeComponent() {
  return (
    <Form action="/projects/123" method="post">
      <input type="text" name="title" />
      <button type="submit">Submit</button>
    </Form>
  );
}
```

**Calling actions with useSubmit**

- `useSubmit`으로 명령형(submit) 호출
- 동일하게 내비게이션을 발생시키며 browser history에 새 항목을 추가한다.

```tsx
import { useCallback } from "react";
import { useSubmit } from "react-router";
import { useFakeTimer } from "fake-lib";

function useQuizTimer() {
  let submit = useSubmit();

  let cb = useCallback(() => {
    submit(
      { quizTimedOut: true },
      { action: "/end-quiz", method: "post" },
    );
  }, []);

  let tenMinutes = 10 * 60 * 1000;
  useFakeTimer(tenMinutes, cb);
}
```

**Calling actions with a fetcher**

- fetcher는 **내비게이션 및 browser history 추가 없이** action을 호출

```tsx
import { useFetcher } from "react-router";

function Task() {
  let fetcher = useFetcher();
  let busy = fetcher.state !== "idle";

  return (
    <fetcher.Form method="post" action="/update-task/123">
      <input type="text" name="title" />
      <button type="submit">
        {busy ? "Saving..." : "Save"}
      </button>
    </fetcher.Form>
  );
}
```

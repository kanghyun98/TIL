# Route Module

### ▶️ Introduction

- `routes.ts`에서 참조된 파일들은 Route Modules라 불린다.

```tsx
route("teams/:teamId", "./team.tsx"),
//           route module ^^^^^^^^
```

- Route Module은 React Router의 프레임워크 기능 기반
- 아래 기능들을 정의한다.
    - automatic code-splitting
    - data loading
    - actions
    - revalidation
    - error boundaries
    - and more

### ▶️ Component (`default`)

라우트 모듈에서 `default`로 export한 함수/컴포넌트는 해당 라우트가 매칭될 때 렌더링되는 컴포넌트를 정의한다.

### ▶️ **Props passed to the Component**

렌더링될 때 **React Router가 자동 생성하는 props**가 전달되며, `Route.ComponentProps` 타입에 정의되어 있다.

- `loaderData`: 이 라우트 모듈의 `loader` 함수가 반환한 데이터
- `actionData`: 이 라우트 모듈의 `action` 함수가 반환한 데이터
- `params`: 라우트 파라미터 객체 (if any)
- `matches`: 현재 라우트 트리에서 매칭된 모든 라우트 정보 배열

이 props들을 사용하면 `useLoaderData()`나 `useParams()` 같은 훅 대신 **직접 props에서 데이터에 접근**할 수 있다. 그리고 props에 자동으로 올바른 **타입 안전**이 보장된다.

```tsx
import type { Route } from "./+types/route-name";

export default function MyRouteComponent({
  loaderData,
  actionData,
  params,
  matches,
}: Route.ComponentProps) {
  return (
    <div>
      <h1>Welcome to My Route with Props!</h1>
      <p>Loader Data: {JSON.stringify(loaderData)}</p>
      <p>Action Data: {JSON.stringify(actionData)}</p>
      <p>Route Parameters: {JSON.stringify(params)}</p>
      <p>Matched Routes: {JSON.stringify(matches)}</p>
    </div>
  );
}

```

### ▶️ `middleware`

- Route middleware는 서버에서 문서 요청과 데이터 요청의 전후에 순차적으로 실행된다.
- 이를 통해 로깅, 인증, 응답 후처리 같은 작업을 한 곳에서 수행할 수 있다.
- `next` 함수는 다음 미들웨어로 실행을 넘긴다.
- 리프(leaf) 라우트에서는 `next` 함수가 해당 네비게이션에 대한 loader/action을 실행한다.

```tsx
// 1. "요청 시작" 로그 출력
// 2. next() 호출 → 다음 단계 실행
// 3. loader/action 실행 완료
// 4. "요청 종료" 로그 출력
export async function middleware({ request }, next) {
  console.log("요청 시작");
  const response = await next(); // 다음 middleware 또는 loader 실행
  console.log("요청 종료");
  return response;
}
```

```tsx
import { redirect } from "react-router";

export async function middleware({ request }, next) {
  const user = await getUser(request);

  if (!user) {
    return redirect("/login"); // next 호출 안 함 → 체인 중단
  }

  return next(); // 인증 통과 → 다음 단계 실행
}
```

`clientMiddleware`

- 클라이언트 측에서 실행되는 `middleware`
- 클라이언트 네비게이션 중 브라우저에서 실행된다.
- client middleware는 서버의 HTTP 요청을 감싸는 구조가 아니기 때문에 `Response`를 반환하지 않는다 (server middleware와의 유일한 차이점)

### ▶️ `loader`

- Route loader는 라우트 컴포넌트가 렌더링되기 전에 데이터를 제공한다.
- 서버 렌더링 또는 사전 렌더링(pre-rendering) 빌드 과정에서만 서버에서 호출된다.

```tsx
export async function loader() {
  return { message: "Hello, world!" };
}

export default function MyRoute({ loaderData }) {
  return <h1>{loaderData.message}</h1>;
}
```

`clientLoader`

브라우저에서만 호출되며, route client loader는 route loader에 추가로, 또는 route loader를 대신하여 라우트 컴포넌트에 데이터를 제공한다.

```tsx
export async function clientLoader({ serverLoader }) {
  // call the server loader
  const serverData = await serverLoader();
  // And/or fetch data on the client
  const data = getDataFromClient();
  // Return the data to expose through useLoaderData()
  return data;
}
```

client loader는 함수에 `hydrate` 속성을 설정하면 서버 렌더링된 페이지의 초기 로드 hydration 과정에 참여할 수 있다.

```tsx
export async function clientLoader() {
  // ...
}
clientLoader.hydrate = true as const;
```

### ▶️ `action`

Route actions allow server-side data mutations with automatic revalidation of all loader data on the page when called from `<Form>`, `useFetcher`, and `useSubmit`.

> [`clientAction` params](https://api.reactrouter.com/v7/types/react-router.ClientActionFunctionArgs)
> 

```tsx
// route("/list", "./list.tsx")
import { Form } from "react-router";
import { TodoList } from "~/components/TodoList";

// this data will be loaded after the action completes...
export async function loader() {
  const items = await fakeDb.getItems();
  return { items };
}

// ...so that the list here is updated automatically
export default function Items({ loaderData }) {
  return (
    <div>
      <List items={loaderData.items} />
      <Form method="post" navigate={false} action="/list">
        <input type="text" name="title" />
        <button type="submit">Create Todo</button>
      </Form>
    </div>
  );
}

export async function action({ request }) {
  const data = await request.formData();
  const todo = await fakeDb.addItem({
    title: data.get("title"),
  });
  return { ok: true };
}

```

### ▶️ **Error Boundary**

다른 route module API에서 예외가 발생하면, 해당 route module의 `ErrorBoundary`가 라우트 컴포넌트 대신 렌더링된다.

```tsx
import {
  isRouteErrorResponse,
  useRouteError,
} from "react-router";

export function ErrorBoundary() {
  const error = useRouteError();

  if (isRouteErrorResponse(error)) {
    return (
      <div>
        <h1>
          {error.status} {error.statusText}
        </h1>
        <p>{error.data}</p>
      </div>
    );
  } else if (error instanceof Error) {
    return (
      <div>
        <h1>Error</h1>
        <p>{error.message}</p>
        <p>The stack trace is:</p>
        <pre>{error.stack}</pre>
      </div>
    );
  } else {
    return <h1>Unknown Error</h1>;
  }
}
```

### ▶️ HydrateFallback

- 초기 페이지 로드에서, 라우트 컴포넌트는 client loader가 완료된 후에야 렌더링된다.
- `HydrateFallback`을 export하면, 라우트 컴포넌트 대신 즉시 해당 fallback이 렌더링될 수 있다.

```tsx
export async function clientLoader() {
  const data = await fakeLoadLocalGameData();
  return data;
}

export function HydrateFallback() {
  return <p>Loading Game...</p>;
}

export default function Component({ loaderData }) {
  return <Game data={loaderData} />;
}
```

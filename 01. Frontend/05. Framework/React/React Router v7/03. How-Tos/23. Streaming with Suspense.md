# Streaming with Suspense

React Suspense를 사용한 스트리밍은 중요하지 않은 데이터를 뒤로 미뤄서 초기 렌더링을 더 빠르게 하고, UI 렌더링이 막히지 않도록 도와줍니다.

이 기능은 loader와 action에서 Promise를 반환하는 방식으로 사용할 수 있습니다.

<br>

## 1. loader에서 Promise 반환하기

React Router는 라우트 컴포넌트를 렌더링하기 전에 route loader를 기다립니다.  
중요하지 않은 데이터 때문에 loader 전체가 막히지 않게 하려면, 해당 데이터는 `await`하지 말고 Promise 자체를 반환하면 됩니다.

```tsx
import type { Route } from "./+types/my-route";

export async function loader({}: Route.LoaderArgs) {
  // 이 값은 await 하지 않습니다
  const nonCriticalData = new Promise((resolve) =>
    setTimeout(() => resolve("non-critical"), 5000),
  );

  const criticalData = await new Promise((resolve) =>
    setTimeout(() => resolve("critical"), 300),
  );

  return { nonCriticalData, criticalData };
}
```

단일 Promise 하나만 바로 반환할 수는 없고, 반드시 key를 가진 객체 형태로 반환해야 합니다.

<br>

## 2. fallback UI와 완료된 UI 렌더링하기

반환한 Promise는 `loaderData`에서 사용할 수 있습니다.  
`<Await>`는 해당 Promise가 끝나기를 기다리고, 그동안 `<Suspense>`는 fallback UI를 렌더링합니다.

```tsx
import * as React from "react";
import { Await } from "react-router";

// [previous code]

export default function MyComponent({
  loaderData,
}: Route.ComponentProps) {
  const { criticalData, nonCriticalData } = loaderData;

  return (
    <div>
      <h1>Streaming example</h1>
      <h2>Critical data value: {criticalData}</h2>

      <React.Suspense fallback={<div>Loading...</div>}>
        <Await resolve={nonCriticalData}>
          {(value) => <h3>Non critical value: {value}</h3>}
        </Await>
      </React.Suspense>
    </div>
  );
}
```

<br>

## 3. With React 19

React 19를 실험적으로 사용하고 있다면 `Await` 대신 `React.use`를 사용할 수 있습니다.  
다만 Suspense fallback이 동작하도록 하려면, Promise를 별도 컴포넌트로 내려서 처리해야 합니다.

```tsx
<React.Suspense fallback={<div>Loading...</div>}>
  <NonCriticalUI p={nonCriticalData} />
</React.Suspense>
```

```tsx
function NonCriticalUI({ p }: { p: Promise<string> }) {
  const value = React.use(p);
  return <h3>Non critical value {value}</h3>;
}
```

## 4. Timeouts

기본적으로 loader와 action은 4950ms가 지나면 아직 끝나지 않은 Promise를 reject합니다.  
이 값은 `entry.server.tsx`에서 `streamTimeout` 숫자 값을 export해서 조절할 수 있습니다.

```ts
// handler 함수에서 보류 중인 Promise를 10초 후 모두 reject
export const streamTimeout = 10_000;
```

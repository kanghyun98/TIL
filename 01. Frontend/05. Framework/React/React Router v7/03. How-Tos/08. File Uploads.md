# File Uploads

> 사용자가 `<input type="file" />`로 파일을 선택해서 서버로 보내면,
> React Router의 action에서 그 파일을 받아 처리할 수 있다.



React Router에서는 파일 업로드를 처리하기 위한 기본적인 패턴을 제공합니다.  
파일 업로드는 일반적인 form 제출과 동일하게 처리되지만, **`multipart/form-data` 인코딩을 사용해야 합니다.**

이 가이드는 다음 두 가지 시나리오를 설명합니다.

- 기본 파일 업로드 구현
- 서버 파일 저장(Local Storage) 구현

<br>

## 1. Basic File Upload

> 파일을 업로드 요청에서 꺼내오는 것

기본 파일 업로드는 다음 단계로 구현할 수 있습니다.

<br>

### 1.1 Setup some routes

먼저 파일 업로드를 처리할 라우트를 구성합니다.

```ts
import {
  type RouteConfig,
  route,
} from "@react-router/dev/routes";

export default [
  route("user/:id", "pages/user-profile.tsx", [
    route("avatar", "api/avatar.tsx"),
  ]),
] satisfies RouteConfig;
```

이 예제에서는 다음과 같은 구조를 사용합니다.

- `user/:id` → 사용자 프로필 페이지
- `user/:id/avatar` → 파일 업로드 API

<br>

### 1.2 Add the form data parser

파일 업로드를 처리하기 위해 `form-data-parser` 패키지를 사용할 수 있습니다.

```bash
npm i @remix-run/form-data-parser
```

이 라이브러리는 `request.formData()`를 감싸는 유틸리티로  
**스트리밍 방식으로 파일 업로드를 처리할 수 있도록 지원합니다.**

<br>

### 1.3 Create a route with an upload action

파일 업로드 처리는 `action`에서 수행합니다.

> - loader는 **읽기**
> - action은 **쓰기 / 변경**

```ts
// pages/user-profile.tsx
import {
  type FileUpload,
  parseFormData,
} from "@remix-run/form-data-parser";

// request 안에 브라우저가 보낸 업로드 데이터가 들어있음
export async function action({ request }) {
  const uploadHandler = async (fileUpload: FileUpload) => {
    if (fileUpload.fieldName === "avatar") {
      // 업로드 파일 처리
    }
  };


  // parseFormData가 그 요청을 해석
  const formData = await parseFormData(
    request,
    uploadHandler,
  );

  const file = formData.get("avatar");
}
```

여기서 `parseFormData`는 각 파일에 대해 `uploadHandler`를 호출합니다.

<br>

### 1.4 Form 구성

파일 업로드를 위해 form에는 반드시 다음 설정이 필요합니다.

- `method="post"`
- `encType="multipart/form-data"`

```tsx
export default function Component() {
  return (
    <form method="post" encType="multipart/form-data">
      <input type="file" name="avatar" />
      <button>Submit</button>
    </form>
  );
}
```

## 2. Local Storage Implementation

> 꺼낸 파일을 실제 저장소(디스크 등)에 저장하는 것
> !브라우저 localStorage가 아님 주의!

파일을 서버에 저장하려면 파일 저장 시스템을 추가로 구성해야 합니다.

<br>

### 2.1 Add the storage package

파일 저장을 위해 `file-storage` 패키지를 사용할 수 있습니다.

```bash
npm i @remix-run/file-storage
```

이 라이브러리는 **파일을 key/value 방식으로 저장하는 인터페이스**를 제공합니다.

<br>

### 2.2 Create a storage configuration

먼저 파일 저장을 위한 설정 파일을 만듭니다.

```ts
import { LocalFileStorage } from "@remix-run/file-storage/local";

export const fileStorage = new LocalFileStorage(
  "./uploads/avatars",
);

export function getStorageKey(userId: string) {
  return `user-${userId}-avatar`;
}
```

여기서는 업로드된 파일을 `uploads/avatars` 디렉토리에 저장합니다.

<br>

### 2.3 Implement the upload handler

이제 `uploadHandler`에서 파일을 저장하도록 구현합니다.

```ts
import {
  type FileUpload,
  parseFormData,
} from "@remix-run/form-data-parser";

import {
  fileStorage,
  getStorageKey,
} from "~/avatar-storage.server";

export async function action({
  request,
  params,
}) {
  async function uploadHandler(fileUpload: FileUpload) {
    if (fileUpload.fieldName === "avatar") {
      const key = getStorageKey(params.userId);

      await fileStorage.set(key, fileUpload);

      return fileUpload;
    }
  }

  const formData = await parseFormData(
    request,
    uploadHandler,
  );

  return formData.get("avatar");
}
```

이 코드는 업로드된 파일을 서버에 저장합니다.

<br>

### 2.4 Add a route to serve the uploaded file

마지막으로 업로드된 파일을 제공하는 라우트를 추가할 수 있습니다.  

이를 통해 사용자는 업로드된 파일을 URL을 통해 접근할 수 있습니다.

# Server Bundles

> 이 기능은 호스팅 제공자 통합을 위해 설계된 고급 기능입니다. 앱을 여러 서버 번들로 컴파일할 때는, 요청을 올바른 번들로 전달하는 커스텀 라우팅 계층이 앱 앞단에 필요합니다.

일반적으로 React Router는 서버 코드를 하나의 번들로 빌드하고, 그 번들은 request handler 함수를 export 합니다. 하지만 라우트 트리를 여러 서버 번들로 나누고, 각 번들이 라우트 일부에 대한 request handler 함수를 노출하도록 하고 싶은 경우가 있습니다. 이를 위해 `react-router.config.ts`는 `serverBundles` 옵션을 지원합니다. 이 옵션은 라우트를 서로 다른 서버 번들에 할당하는 함수입니다.

`serverBundles` 함수는 트리의 각 라우트에 대해 호출되며, 해당 라우트를 어떤 서버 번들에 할당할지 나타내는 서버 번들 ID를 반환합니다. 주소 지정이 불가능한 라우트(예: pathless layout route)는 제외됩니다. 이 번들 ID는 서버 빌드 디렉터리 안에서 디렉터리 이름으로 사용됩니다.

각 라우트에 대해 이 함수는 해당 라우트까지 이어지는 라우트 배열을 받는데, 이를 라우트 `branch` 라고 합니다. 이를 통해 라우트 트리의 서로 다른 부분에 대해 서버 번들을 만들 수 있습니다. 예를 들어, 특정 layout route 내부의 모든 라우트를 별도 서버 번들로 만들 수 있습니다.

<br>

~~~ts
import type { Config } from "@react-router/dev/config";

export default {
  // ...
  serverBundles: ({ branch }) => {
    const isAuthenticatedRoute = branch.some((route) =>
      route.id.split("/").includes("_authenticated"),
    );

    return isAuthenticatedRoute
      ? "authenticated"
      : "unauthenticated";
  },
} satisfies Config;
~~~

<br>

`branch` 배열의 각 `route` 는 다음 속성을 가집니다:

- `id` — 이 라우트의 고유 ID입니다. `file` 과 비슷한 이름을 가지지만 app 디렉터리를 기준으로 한 상대 경로이며 확장자는 제외됩니다. 예를 들어 `app/routes/gists.$username.tsx` 의 `id` 는 `routes/gists.$username` 입니다.
- `path` — 이 라우트가 URL pathname 과 매칭할 때 사용하는 경로입니다.
- `file` — 이 라우트의 엔트리 포인트에 대한 절대 경로입니다.
- `index` — 이 라우트가 index route 인지 여부입니다.

## 1. Build manifest

빌드가 완료되면 React Router는 `buildEnd` 훅을 호출하면서 `buildManifest` 객체를 전달합니다. 이 객체는 요청을 올바른 서버 번들로 라우팅하는 방법을 결정하기 위해 build manifest 를 확인해야 할 때 유용합니다.

<br>

~~~ts
import type { Config } from "@react-router/dev/config";

export default {
  // ...
  buildEnd: async ({ buildManifest }) => {
    // ...
  },
} satisfies Config;
~~~

<br>

서버 번들을 사용할 때 build manifest 는 다음 속성을 포함합니다:

- `serverBundles` — 번들 ID를 해당 번들의 `id` 와 `file` 에 매핑하는 객체입니다.
- `routeIdToServerBundleId` — 라우트 ID를 해당 서버 번들 ID에 매핑하는 객체입니다.
- `routes` — 라우트 ID를 라우트 메타데이터에 매핑하는 route manifest 입니다. 이것은 React Router request handler 앞단의 커스텀 라우팅 계층을 구동하는 데 사용할 수 있습니다.

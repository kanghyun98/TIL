# Presets

React Router config는 다른 도구 및 호스팅 환경과의 통합을 쉽게 하기 위해 `presets` 옵션을 지원합니다.

Presets는 다음 두 가지 역할만 수행합니다:

- React Router config 옵션을 대신 설정
- 최종 config를 검증

각 preset에서 반환된 config는 정의된 순서대로 병합되며, 사용자가 직접 작성한 config는 마지막에 병합됩니다. 따라서 사용자 config가 항상 우선합니다.

> - 직접 config 작성 → “수동 설정”
> - preset 사용 → “템플릿/자동 설정”

<br>

## 1. Defining preset config

기본적인 예시로 server bundles 함수를 설정하는 preset을 만들 수 있습니다.

```ts
import type { Preset } from "@react-router/dev/config";

export function myCoolPreset(): Preset {
  return {
    name: "my-cool-preset",
    reactRouterConfig: () => ({
      serverBundles: ({ branch }) => {
        const isAuthenticatedRoute = branch.some((route) =>
          route.id.split("/").includes("_authenticated"),
        );

        return isAuthenticatedRoute
          ? "authenticated"
          : "unauthenticated";
      },
    }),
  };
}
```

<br>

## 2. Validating config

다른 preset이나 user config가 preset 값을 덮어쓸 수 있다는 점을 고려해야 합니다.

예를 들어 `serverBundles`가 변경되지 않도록 보장하려면 `reactRouterConfigResolved` 훅을 사용할 수 있습니다.

```ts
import type {
  Preset,
  ServerBundlesFunction,
} from "@react-router/dev/config";

const serverBundles: ServerBundlesFunction = ({
  branch,
}) => {
  const isAuthenticatedRoute = branch.some((route) =>
    route.id.split("/").includes("_authenticated"),
  );

  return isAuthenticatedRoute
    ? "authenticated"
    : "unauthenticated";
};

export function myCoolPreset(): Preset {
  return {
    name: "my-cool-preset",
    reactRouterConfig: () => ({ serverBundles }),
    // here!
    reactRouterConfigResolved: ({ reactRouterConfig }) => {
      if (
        reactRouterConfig.serverBundles !== serverBundles
      ) {
        throw new Error("`serverBundles` was overridden!");
      }
    },
  };
}
```

<br>

## 3. Using a preset

Presets는 npm 패키지로 배포되어 React Router config에서 사용할 수 있도록 설계되었습니다.

```ts
// react-router.config.ts
import type { Config } from "@react-router/dev/config";
import { myCoolPreset } from "react-router-preset-cool";

export default {
  // ...
  presets: [myCoolPreset()],
} satisfies Config;
```

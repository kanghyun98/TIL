# Navigation Blocking

사용자가 중요한 작업(예: 폼 입력)을 진행 중일 때, 페이지를 벗어나지 못하도록 navigation을 차단할 수 있습니다. 

이 가이드는 다음 내용을 다룹니다:

- fetcher를 사용하는 form과 action 설정
- form이 dirty 상태일 때 navigation 차단
- 페이지 이탈 시 confirmation UI 표시

<br>

## 1. Set up a route with a form

form이 있는 route를 설정합니다. 예제에서는 `contact` route를 사용합니다.

```ts
import {
  type RouteConfig,
  index,
  route,
} from "@react-router/dev/routes";

export default [
  index("routes/home.tsx"),
  route("contact", "routes/contact.tsx"),
] satisfies RouteConfig;
```

<br>

contact route module에 form을 추가합니다.

```ts
import { useFetcher } from "react-router";
import type { Route } from "./+types/contact";

export async function action({
  request,
}: Route.ActionArgs) {
  let formData = await request.formData();
  let email = formData.get("email");
  let message = formData.get("message");
  console.log(email, message);
  return { ok: true };
}

export default function Contact() {
  let fetcher = useFetcher();

  return (
    <fetcher.Form method="post">
      <p>
        <label>
          Email: <input name="email" type="email" />
        </label>
      </p>
      <p>
        <textarea name="message" />
      </p>
      <p>
        <button type="submit">
          {fetcher.state === "idle" ? "Send" : "Sending..."}
        </button>
      </p>
    </fetcher.Form>
  );
}
```

<br>

## 2. Add dirty state and onChange handler

form의 변경 여부를 추적하기 위해 boolean 상태를 사용합니다.

```ts
export default function Contact() {
  let [isDirty, setIsDirty] = useState(false);
  let fetcher = useFetcher();

  return (
    <fetcher.Form
      method="post"
      onChange={(event) => {
        let email = event.currentTarget.email.value;
        let message = event.currentTarget.message.value;
        setIsDirty(Boolean(email || message));
      }}
    >
      {/* existing code */}
    </fetcher.Form>
  );
}
```

<br>

## 3. Block navigation when the form is dirty

`useBlocker`를 사용하여 조건에 따라 navigation을 차단합니다.

```ts
import { useBlocker } from "react-router";

export default function Contact() {
  let [isDirty, setIsDirty] = useState(false);
  let fetcher = useFetcher();
  let blocker = useBlocker(
    useCallback(() => isDirty, [isDirty]),
  );

  // ... existing code
}
```

navigation은 차단되지만, 아직 사용자가 선택할 수 있는 UI는 없습니다.

<br>

## 4. Show confirmation UI

사용자가 페이지를 떠날지 결정할 수 있도록 UI를 표시합니다.

```ts
export default function Contact() {
  let [isDirty, setIsDirty] = useState(false);
  let fetcher = useFetcher();
  let blocker = useBlocker(
    useCallback(() => isDirty, [isDirty]),
  );

  return (
    <fetcher.Form
      method="post"
      onChange={(event) => {
        let email = event.currentTarget.email.value;
        let message = event.currentTarget.message.value;
        setIsDirty(Boolean(email || message));
      }}
    >
      {/* existing code */}

      {blocker.state === "blocked" && (
        <div>
          <p>Wait! You didn't send the message yet:</p>
          <p>
            <button
              type="button"
              onClick={() => blocker.proceed()}
            >
              Leave
            </button>{" "}
            <button
              type="button"
              onClick={() => blocker.reset()}
            >
              Stay here
            </button>
          </p>
        </div>
      )}
    </fetcher.Form>
  );
}
```

- `blocker.proceed()` → navigation 진행
- `blocker.reset()` → 현재 페이지 유지

<br>

## 5. Reset the blocker when the action resolves

사용자가 버튼을 누르지 않고 form을 제출하면 blocker가 여전히 활성 상태로 남습니다. 이를 해결하기 위해 action 완료 시 blocker를 초기화합니다.

```ts
useEffect(() => {
  if (fetcher.data?.ok) {
    if (blocker.state === "blocked") {
      blocker.reset();
    }
  }
}, [fetcher.data]);
```

<br>

## 6. Clear the form when the action resolves

form 제출 후 입력값을 초기화할 수 있습니다.

```ts
let formRef = useRef<HTMLFormElement>(null);

<fetcher.Form
  ref={formRef}
  method="post"
  onChange={(event) => {
    // ... existing code
  }}
>
  {/* existing code */}
</fetcher.Form>;

useEffect(() => {
  if (fetcher.data?.ok) {
    formRef.current?.reset();
    if (blocker.state === "blocked") {
      blocker.reset();
    }
  }
}, [fetcher.data]);
```

또는 blocker를 reset하는 대신 navigation을 진행할 수도 있습니다.

```ts
useEffect(() => {
  if (fetcher.data?.ok) {
    if (blocker.state === "blocked") {
      blocker.proceed();
    } else {
      formRef.current?.reset();
    }
  }
}, [fetcher.data]);
```

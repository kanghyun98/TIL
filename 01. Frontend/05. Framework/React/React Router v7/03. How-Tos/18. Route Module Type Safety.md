# Route Module Type Safety

React Router는 URL 파라미터, loader 데이터 등을 위한 타입 추론이 가능하도록 라우트별 타입을 생성합니다. 이 글은 템플릿으로 시작하지 않은 경우 이를 설정하는 방법을 설명합니다.

타입 안전성이 어떻게 동작하는지는 `Type Safety Explanation`에서 더 자세히 확인할 수 있습니다.

<br>

## 1. `.react-router/`를 `.gitignore`에 추가하기

React Router는 앱 루트에 `.react-router/` 디렉터리를 만들고 여기에 타입을 생성합니다. 이 디렉터리는 React Router가 전적으로 관리하므로 `.gitignore`에 추가해야 합니다.

```txt
.react-router/
```

<br>

## 2. 생성된 타입을 `tsconfig`에 포함하기

TypeScript가 생성된 타입을 사용할 수 있도록 `tsconfig`를 수정합니다. 또한 타입을 라우트 모듈의 상대적인 형제 경로처럼 가져올 수 있도록 `rootDirs`도 설정해야 합니다.

```json
{
  "include": [".react-router/types/**/*"],
  "compilerOptions": {
    "rootDirs": [".", "./.react-router/types"]
  }
}
```

앱에서 여러 개의 `tsconfig` 파일을 사용하고 있다면, 이 변경은 앱 디렉터리를 `include`하는 파일에 적용해야 합니다.

예를 들어 `node-custom-server` 템플릿에는 `tsconfig.json`, `tsconfig.node.json`, `tsconfig.vite.json`이 있습니다. 이 중 앱 디렉터리를 `include`하는 파일이 `tsconfig.vite.json`이라면, 라우트 모듈 타입 안전성을 위한 `.react-router/types` 설정도 그 파일에 해야 합니다.

<br>

## 3. 타입 체크 전에 타입 생성하기

타입 체크를 별도 명령으로 실행하고 싶다면, 예를 들어 Continuous Integration 파이프라인의 일부로 사용한다면, 타입 체크를 실행하기 전에 반드시 타입을 먼저 생성해야 합니다.

```json
{
  "scripts": {
    "typecheck": "react-router typegen && tsc"
  }
}
```

## 5.  `Context` 타입 확장

앱의 `context` 타입을 정의하려면, 프로젝트 안의 `.ts` 또는 `.d.ts` 파일에 다음 내용을 추가합니다.

```ts
import "react-router";

declare module "react-router" {
  interface AppLoadContext {
    // add context properties here
  }
}
```

<br>

## 6. Type-only auto-imports (optional)

`Route` 타입 헬퍼를 자동 import하면 TypeScript는 다음과 같이 생성합니다.

```ts
import { Route } from "./+types/my-route";
```

하지만 `verbatimModuleSyntax`를 활성화하면:

```json
{
  "compilerOptions": {
    "verbatimModuleSyntax": true
  }
}
```

다음처럼 `type` 한정자도 자동으로 함께 생성됩니다.

```ts
import type { Route } from "./+types/my-route";
//     ^^^^
```

이렇게 하면 bundler 같은 도구가 타입 전용 모듈을 감지하고, 번들에서 안전하게 제외할 수 있습니다.

<br>

## 7. 결론

React Router의 Vite 플러그인은 라우트 설정인 `routes.ts`를 수정할 때마다 `.react-router/types/`에 타입을 자동으로 생성해야 합니다. 즉, 라우트의 최신 타입을 유지하려면 `react-router dev` 또는 사용자 정의 개발 서버를 실행하면 됩니다.

이 타입들을 라우트에서 가져오는 예시는 `Type Safety Explanation`에서 확인할 수 있습니다.

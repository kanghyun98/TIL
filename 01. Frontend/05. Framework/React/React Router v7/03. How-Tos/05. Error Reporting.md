# Error Reporting

React Router는 라우트 모듈에서 발생한 오류를 자동으로 감지하고 가장 가까운 **ErrorBoundary**로 전달합니다.  
이를 통해 오류가 발생하더라도 애플리케이션이 빈 화면을 렌더링하는 상황을 방지할 수 있습니다.

하지만 ErrorBoundary만으로는 오류를 **로깅하거나 외부 서비스로 보고하는 기능**을 제공하지 않습니다.  
따라서 서버와 클라이언트 각각에서 오류를 수집하고 보고하는 방식을 추가로 구성할 수 있습니다.

<br>

## 1. Server Errors

서버에서 발생한 오류를 처리하려면 **서버 엔트리 모듈에서 `handleError` 함수를 export**해야 합니다.

### 1.1 Server entry 파일 생성

프로젝트에 `entry.server.tsx` 파일이 없다면 기본 서버 엔트리를 사용하고 있는 상태입니다.  
다음 CLI 명령을 실행하면 기본 server entry 파일을 프로젝트에 생성할 수 있습니다.

```
react-router reveal entry.server
```

<br>

### 1.2 Error handler 추가

`handleError` 함수는 React Router가 서버에서 오류를 감지할 때 호출됩니다.

```ts
import { type HandleErrorFunction } from "react-router";

export const handleError: HandleErrorFunction = (
  error,
  { request },
) => {
  // React Router는 중단된 요청도 발생시킬 수 있으므로
  // abort된 요청은 로그하지 않는 것이 좋습니다
  if (!request.signal.aborted) {
    myReportError(error);

    // 콘솔에도 출력하여 오류 확인 가능
    console.error(error);
  }
};
```

이 함수를 사용하면 다음과 같은 작업을 수행할 수 있습니다.

- 서버 오류 로깅
- 오류 리포팅 서비스 연동
- 서버 모니터링 시스템 연동

<br>

## 2. Client Errors

클라이언트에서 발생한 오류에 접근하려면  
`HydratedRouter` 또는 `RouterProvider`의 `onError` prop을 사용할 수 있습니다.

<br>

### 2.1 Client entry 파일 생성

프로젝트에 `entry.client.tsx` 파일이 없다면 기본 클라이언트 엔트리를 사용하고 있는 상태입니다.

다음 명령어로 파일을 생성할 수 있습니다.

```
react-router reveal entry.client
```

<br>

### 2.2 Client error handler 추가

클라이언트에서 오류를 처리하려면 `onError` 함수를 정의하고 라우터에 전달합니다.

```ts
import { type ClientOnErrorFunction } from "react-router";

const onError: ClientOnErrorFunction = (
  error,
  { location, params, unstable_pattern, errorInfo },
) => {
  myReportError(error, location, errorInfo);

  console.error(error, errorInfo);
};
```

Framework Mode에서는 `HydratedRouter`에 전달합니다.

```ts
startTransition(() => {
  hydrateRoot(
    document,
    <StrictMode>
      <HydratedRouter onError={onError} />
    </StrictMode>
  );
});
```

<br>

## 3. ErrorBoundary와 Error Reporting의 차이

React Router에서 오류 처리는 두 가지 역할로 나눌 수 있습니다.

### ErrorBoundary

- 사용자에게 오류 UI를 보여주기 위한 기능
- 라우트 단위로 렌더링

### Error Reporting

- 오류를 로깅하거나 외부 서비스로 보고하기 위한 기능
- 서버 또는 클라이언트에서 처리

일반적으로 두 기능을 함께 사용하는 것이 좋습니다.

```
ErrorBoundary → 사용자에게 오류 UI 표시  
Error Reporting → 로그 및 모니터링
```

# Using Fetchers

Fetchers는 **네비게이션을 발생시키지 않고 데이터를 로드하거나 변경할 수 있도록 해주는 도구**입니다.

일반적인 라우트 네비게이션과 달리 fetcher는 **URL을 변경하지 않으면서 loader나 action을 호출할 수 있습니다.**  
따라서 복잡하고 동적인 사용자 인터페이스를 구현할 때 매우 유용합니다.

예를 들어 다음과 같은 상황에서 사용됩니다.

- 리스트 항목 수정
- 좋아요 버튼
- 자동완성 검색
- 개별 필드 업데이트
- 삭제 버튼

이처럼 **페이지 이동 없이 서버와 상호작용해야 하는 UI**에 적합합니다.

<br>

## 1. Calling Actions

fetcher의 가장 일반적인 사용 사례는 **action에 데이터를 제출하는 것**입니다.

fetcher를 사용해 action을 호출하면 다음과 같은 흐름이 발생합니다.

1. 폼 제출
2. action 실행
3. 데이터 재검증
4. UI 업데이트

<br>

### 1.1 Add an action

먼저 fetcher가 호출할 action을 라우트에 추가합니다.

```tsx
export async function clientLoader() {
  let title = localStorage.getItem("title") || "No Title";
  return { title };
}

export async function clientAction({ request }) {
  await new Promise((res) => setTimeout(res, 1000));
  let data = await request.formData();
  localStorage.setItem("title", data.get("title"));

  return { ok: true };
}
```

<br>

### 1.2 Create a fetcher

다음으로 `useFetcher()`를 사용해 fetcher를 생성합니다.

```tsx
import { useLoaderData, useFetcher } from "react-router";

export default function Component() {
  let data = useLoaderData();
  let fetcher = useFetcher();

  return (
    <div>
      <h1>{data.title}</h1>

      <fetcher.Form method="post">
        <input type="text" name="title" />
      </fetcher.Form>
    </div>
  );
}
```

`fetcher.Form`은 일반 `<Form>`과 비슷하지만  
**페이지 이동 없이 action을 호출**합니다.

<br>

### 1.3 Submit the form

사용자가 폼을 제출하면 fetcher가 action을 호출합니다.

이때 React Router는 action이 완료된 뒤  
관련된 loader 데이터를 자동으로 **revalidation** 합니다.

<br>

### 1.4 Render pending state

fetcher는 자체 상태를 관리합니다.

```tsx
let busy = fetcher.state !== "idle";
```

이를 활용하면 다음과 같이 pending UI를 구현할 수 있습니다.

```tsx
<button type="submit">
  {busy ? "Saving..." : "Save"}
</button>
```

fetcher 상태는 다음 값을 가질 수 있습니다.

- `"idle"`
- `"submitting"`
- `"loading"`

<br>

### 1.5 Optimistic UI

fetcher는 **optimistic UI 패턴을 구현할 때도 사용할 수 있습니다.**

즉 서버 응답을 기다리지 않고 **UI를 먼저 업데이트**하는 방식입니다.

이 패턴은 사용자 경험을 개선할 때 자주 사용됩니다.

<br>

### 1.6 Fetcher Data and Validation

action이 완료되면 fetcher는 결과 데이터를 제공합니다.

```tsx
fetcher.data
```

이 데이터를 활용하면 다음 작업을 할 수 있습니다.

- UI 업데이트
- validation 메시지 표시
- 오류 처리

<br>

## 2. Loading Data

fetcher는 데이터를 변경하는 것뿐 아니라 **데이터를 로드하는 데에도 사용할 수 있습니다.**

예를 들어 검색 UI를 구현할 때 사용할 수 있습니다.

<br>

### 2.1 Create a search route

먼저 검색을 처리할 라우트를 생성합니다.

```tsx
// { path: '/search-users', filename: './search-users.tsx' }
const users = [
  { id: 1, name: "Ryan" },
  { id: 2, name: "Michael" },
  // ...
];

export async function loader({ request }) {
  await new Promise((res) => setTimeout(res, 300));
  let url = new URL(request.url);
  let query = url.searchParams.get("q");
  return users.filter((user) =>
    user.name.toLowerCase().includes(query.toLowerCase()),
  );
}
```



<br>

### 2.2 Render a fetcher in a combobox component

다음으로 fetcher를 사용해 검색 UI를 구현합니다.

```tsx
import { useFetcher } from "react-router";

export function UserSearchCombobox() {
  let fetcher = useFetcher();

  return (
    <div>
      <fetcher.Form method="get" action="/search-users">
        <input type="text" name="q" />
      </fetcher.Form>
    </div>
  );
}
```

<br>

### 2.3 Add type inference

fetcher는 loader 타입을 활용해 타입 추론을 사용할 수 있습니다.

```tsx
let fetcher = useFetcher<typeof loader>();
```

<br>

### 2.4 Render the data

검색 결과는 `fetcher.data`를 통해 접근할 수 있습니다.

```tsx
{fetcher.data && (
  <ul>
    {fetcher.data.map((user) => (
      <li key={user.id}>{user.name}</li>
    ))}
  </ul>
)}
```

<br>

### 2.5 Render a pending state

검색 요청이 진행 중일 때는 다음과 같이 상태를 활용할 수 있습니다.

```tsx
fetcher.state === "loading"
```

<br>

### 2.6 Search on user input

fetcher는 사용자 입력 이벤트와 결합해  
**자동 검색 UI** 같은 인터랙션을 구현할 때 유용합니다.

예를 들어 입력값이 변경될 때마다  
검색 요청을 보내는 방식입니다.

<br>

## 3. When to use Fetchers

fetcher는 다음과 같은 상황에서 특히 유용합니다.

- URL을 변경하지 않는 데이터 변경
- 리스트 항목 수정
- 개별 필드 업데이트
- 삭제 버튼
- 검색 UI
- 자동완성 UI

이러한 작업은 보통 **페이지 이동 없이 현재 UI에서 처리되기를 기대하기 때문입니다.**

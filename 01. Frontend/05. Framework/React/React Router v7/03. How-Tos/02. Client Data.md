# React Router Client Data

React Router에서는 브라우저에서 직접 데이터를 가져오거나 변경할 수 있습니다.  
이를 위해 `clientLoader`와 `clientAction` 함수를 사용할 수 있습니다.

이 함수들은 특히 **SPA 환경에서 데이터를 처리하기 위한 주요 메커니즘**으로 사용됩니다.  
또한 서버 렌더링 환경에서도 클라이언트 데이터를 함께 활용할 수 있습니다.

<br>

## 1. Skip the Server Hop

React Router를 Backend-For-Frontend(BFF) 아키텍처와 함께 사용할 때  
React Router 서버를 거치지 않고 **백엔드 API와 직접 통신**하고 싶을 수 있습니다.

이 접근 방식은 다음 조건을 전제로 합니다.

- 적절한 인증 처리
- CORS 제한 없음

이 패턴의 기본 구현 방식은 다음과 같습니다.

1. 문서(document) 로드 시 서버 `loader`에서 데이터를 로드  
2. 이후 네비게이션에서는 `clientLoader`에서 데이터를 로드  

```ts
export async function loader({ request }) {
  const data = await fetchApiFromServer({ request }); // (1)
  return data;
}

export async function clientLoader({ request }) {
  const data = await fetchApiFromClient({ request }); // (2)
  return data;
}
```

이 경우 hydration 중에는 `clientLoader`가 실행되지 않고  
이후 네비게이션에서만 실행됩니다.

<br>

## 2. Fullstack State

컴포넌트를 렌더링하기 전에 **서버 데이터와 브라우저 데이터**를 함께 사용해야 하는 경우가 있습니다.

예를 들면 다음과 같습니다.

- 서버 데이터베이스
- IndexedDB
- 브라우저 SDK

이 패턴은 다음 단계로 구현할 수 있습니다.

1. 서버 `loader`에서 부분 데이터를 로드  
2. SSR 중에는 `HydrateFallback` UI 렌더링  
3. `clientLoader.hydrate = true` 설정  
4. `clientLoader`에서 서버 데이터와 클라이언트 데이터를 결합  

```ts
export async function loader({ request }) {
  const partialData = await getPartialDataFromDb({ request }); // (1)
  return partialData;
}

export async function clientLoader({ request, serverLoader }) {
  const [serverData, clientData] = await Promise.all([
    serverLoader(),
    getClientData(request),
  ]);

  return {
    ...serverData,
    ...clientData,
  };
}

clientLoader.hydrate = true;

export function HydrateFallback() {
  return <p>Skeleton rendered during SSR</p>;
}

export default function Component({ loaderData }) {
  return <>...</>;
}
```

이 방식은 **SSR 데이터와 브라우저 데이터를 동시에 활용할 때 유용한 패턴**입니다.

<br>

## 3. Choosing Server or Client Data Loading

React Router에서는 **라우트마다 데이터 로딩 전략을 선택할 수 있습니다.**

예를 들어

- 어떤 라우트는 서버 데이터를 사용하고
- 어떤 라우트는 클라이언트 데이터를 사용할 수 있습니다.

### 서버 loader만 사용하는 경우

```ts
export async function loader({ request }) {
  const data = await getServerData(request);
  return data;
}

export default function Component({ loaderData }) {
  return <>...</>;
}
```

### 클라이언트 loader만 사용하는 경우

```ts
export async function clientLoader({ request }) {
  const clientData = await getClientData(request);
  return clientData;
}

clientLoader.hydrate = true;

export function HydrateFallback() {
  return <p>Skeleton rendered during SSR</p>;
}

export default function Component({ loaderData }) {
  return <>...</>;
}
```

이처럼 React Router에서는 **각 라우트마다 다른 데이터 전략을 사용할 수 있습니다.**

<br>

## 4. Client-Side Caching

클라이언트 캐싱을 사용하면 **서버 요청을 줄이고 성능을 개선할 수 있습니다.**

일반적인 흐름은 다음과 같습니다.

1. 문서 로드 시 서버 `loader` 실행  
2. `clientLoader.hydrate = true`로 캐시 초기화  
3. 이후 네비게이션에서는 캐시 데이터를 사용  
4. `clientAction`에서 캐시 무효화  

```ts
export async function loader({ request }) {
  const data = await getDataFromDb({ request }); // (1)
  return data;
}

export async function action({ request }) {
  await saveDataToDb({ request });
  return { ok: true };
}

let isInitialRequest = true;

export async function clientLoader({ request, serverLoader }) {
  const cacheKey = generateKey(request);

  if (isInitialRequest) {
    isInitialRequest = false;
    const serverData = await serverLoader();
    cache.set(cacheKey, serverData); // (2)
    return serverData;
  }

  const cachedData = await cache.get(cacheKey);
  if (cachedData) {
    return cachedData; // (3)
  }

  const serverData = await serverLoader();
  cache.set(cacheKey, serverData);
  return serverData;
}

clientLoader.hydrate = true;

export async function clientAction({ request, serverAction }) {
  const cacheKey = generateKey(request);
  cache.delete(cacheKey); // (4)
  const serverData = await serverAction();
  return serverData;
}
```

<br>

## 5. Hydration 주의사항

`HydrateFallback`을 export하지 않는 경우에는 주의해야 합니다.

SSR에서 라우트 컴포넌트를 렌더링한 후 hydration 단계에서 `clientLoader`가 실행되기 때문에  
**초기 로드 시 `loader`와 `clientLoader`가 동일한 데이터를 반환해야 합니다.**

그렇지 않으면 **hydration mismatch 오류가 발생할 수 있습니다.**

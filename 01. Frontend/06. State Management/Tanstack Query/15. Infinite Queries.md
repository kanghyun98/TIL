# Infinite Queries

기존 데이터에 다음 데이터를 계속 덧붙여 보여주는 리스트, 즉 `"load more"` 또는 무한 스크롤은 매우 흔한 UI 패턴입니다. TanStack Query는 이런 리스트를 조회하기 위해 `useQuery`의 확장 형태인 `useInfiniteQuery`를 제공합니다.

`useInfiniteQuery`를 사용할 때는 몇 가지가 달라집니다.

- `data`는 이제 무한 쿼리 데이터를 담는 객체입니다.
  - `data.pages`: 가져온 페이지 목록
  - `data.pageParams`: 각 페이지를 가져올 때 사용한 page param 목록
- `fetchNextPage`, `fetchPreviousPage` 함수를 사용할 수 있습니다.
  - `fetchNextPage`는 필수입니다.
- `initialPageParam` 옵션을 사용할 수 있습니다.
  - 반드시 지정해야 합니다.
- `getNextPageParam`, `getPreviousPageParam` 옵션을 사용할 수 있습니다.
  - 더 불러올 데이터가 있는지 판단합니다.
  - 다음 또는 이전 페이지를 가져오는 데 필요한 정보도 제공합니다.
  - 이 정보는 query function에 추가 파라미터로 전달됩니다.
- `hasNextPage`, `hasPreviousPage` 를 사용할 수 있습니다.
  - `getNextPageParam`, `getPreviousPageParam`이 `null` 또는 `undefined`가 아닌 값을 반환하면 `true`입니다.
- `isFetchingNextPage`, `isFetchingPreviousPage`를 사용할 수 있습니다.
  - 백그라운드 새로고침 상태와 추가 페이지 로딩 상태를 구분할 수 있습니다.

> 참고: `initialData` 또는 `placeholderData`는 `data.pages`, `data.pageParams`를 가진 동일한 구조의 객체여야 합니다.

<br>

## 1. Example

프로젝트를 3개씩 페이지 단위로 반환하고, 다음 페이지를 가져올 때 사용할 cursor를 함께 내려주는 API를 가정합니다.

```tsx
fetch('/api/projects?cursor=0')
// { data: [...], nextCursor: 3}
fetch('/api/projects?cursor=3')
// { data: [...], nextCursor: 6}
fetch('/api/projects?cursor=6')
// { data: [...], nextCursor: 9}
fetch('/api/projects?cursor=9')
// { data: [...] }
```

이 정보를 바탕으로 `"Load More"` UI를 만들 수 있습니다.

- 기본적으로 `useInfiniteQuery`가 첫 번째 데이터 그룹을 요청하도록 둡니다.
- `getNextPageParam`에서 다음 요청에 필요한 정보를 반환합니다.
- `fetchNextPage`를 호출합니다.

```tsx
import { useInfiniteQuery } from '@tanstack/react-query'

function Projects() {
  const fetchProjects = async ({ pageParam }) => {
    const res = await fetch('/api/projects?cursor=' + pageParam)
    return res.json()
  }

  const {
    data,
    error,
    fetchNextPage,
    hasNextPage,
    isFetching,
    isFetchingNextPage,
    status,
  } = useInfiniteQuery({
    queryKey: ['projects'],
    queryFn: fetchProjects,
    initialPageParam: 0,
    getNextPageParam: (lastPage, pages) => lastPage.nextCursor,
  })

  return status === 'pending' ? (
    <p>Loading...</p>
  ) : status === 'error' ? (
    <p>Error: {error.message}</p>
  ) : (
    <>
      {data.pages.map((group, i) => (
        <React.Fragment key={i}>
          {group.data.map((project) => (
            <p key={project.id}>{project.name}</p>
          ))}
        </React.Fragment>
      ))}
      <div>
        <button
          onClick={() => fetchNextPage()}
          disabled={!hasNextPage || isFetching}
        >
          {isFetchingNextPage
            ? 'Loading more...'
            : hasNextPage
              ? 'Load More'
              : 'Nothing more to load'}
        </button>
      </div>
      <div>{isFetching && !isFetchingNextPage ? 'Fetching...' : null}</div>
    </>
  )
}
```

`fetchNextPage`를 호출할 때 이미 fetch가 진행 중이면, 백그라운드에서 일어나는 데이터 새로고침을 덮어쓸 위험이 있습니다. 특히 리스트를 렌더링하면서 동시에 `fetchNextPage`를 트리거하는 경우 이 문제가 더 중요해집니다.

Infinite Query에서는 동시에 진행될 수 있는 fetch가 하나뿐이라는 점을 이해해야 합니다. 모든 페이지는 하나의 캐시 엔트리를 공유하므로, 동시에 두 번 fetch를 시도하면 데이터가 덮어써질 수 있습니다.

동시 fetch를 허용하려면 `fetchNextPage`에 `{ cancelRefetch: false }` 옵션을 사용할 수 있습니다. 기본값은 `true`입니다.

충돌 없이 쿼리를 처리하려면, 특히 사용자가 직접 호출을 제어하지 않는 상황에서는 쿼리가 `isFetching` 상태가 아닌지 확인하는 것이 권장됩니다.

```jsx
<List onEndReached={() => hasNextPage && !isFetching && fetchNextPage()} />
```

## 2. What happens when an infinite query needs to be refetched?

무한 쿼리가 stale 상태가 되어 다시 fetch되어야 하면, 각 그룹은 첫 번째 그룹부터 순차적으로 fetch됩니다. 이렇게 해야 원본 데이터가 변경되었더라도 오래된 cursor를 사용해 중복 데이터를 가져오거나 일부 레코드를 건너뛰는 문제를 막을 수 있습니다.

무한 쿼리 결과가 `queryCache`에서 제거되면, 페이지네이션은 초기 상태로 다시 시작되고 첫 번째 그룹만 다시 요청됩니다.

<br>

## 3. What if I want to implement a bi-directional infinite list?

양방향 리스트는 다음 속성과 함수를 사용해 구현할 수 있습니다.

- `getPreviousPageParam`
- `fetchPreviousPage`
- `hasPreviousPage`
- `isFetchingPreviousPage`

```tsx
useInfiniteQuery({
  queryKey: ['projects'],
  queryFn: fetchProjects,
  initialPageParam: 0,
  getNextPageParam: (lastPage, pages) => lastPage.nextCursor,
  getPreviousPageParam: (firstPage, pages) => firstPage.prevCursor,
})
```

<br>

## 4. What if I want to show the pages in reversed order?

페이지를 역순으로 보여주고 싶다면 `select` 옵션을 사용할 수 있습니다.

```tsx
useInfiniteQuery({
  queryKey: ['projects'],
  queryFn: fetchProjects,
  select: (data) => ({
    pages: [...data.pages].reverse(),
    pageParams: [...data.pageParams].reverse(),
  }),
})
```

<br>

## 5. What if I want to manually update the infinite query?

### 5.1. Manually removing first page

```tsx
queryClient.setQueryData(['projects'], (data) => ({
  pages: data.pages.slice(1),
  pageParams: data.pageParams.slice(1),
}))
```

### 5.2. Manually removing a single value from an individual page

```tsx
const newPagesArray =
  oldPagesArray?.pages.map((page) =>
    page.filter((val) => val.id !== updatedId),
  ) ?? []

queryClient.setQueryData(['projects'], (data) => ({
  pages: newPagesArray,
  pageParams: data.pageParams,
}))
```

### 5.3. Keep only the first page

```tsx
queryClient.setQueryData(['projects'], (data) => ({
  pages: data.pages.slice(0, 1),
  pageParams: data.pageParams.slice(0, 1),
}))
```

항상 `pages`와 `pageParams`의 데이터 구조를 동일하게 유지해야 합니다.

<br>

## 6. What if I want to limit the number of pages?

일부 경우에는 성능과 UX를 위해 쿼리 데이터에 저장되는 페이지 수를 제한하고 싶을 수 있습니다.

- 사용자가 매우 많은 페이지를 불러올 수 있는 경우
  - 메모리 사용량 문제
- 수십 개 페이지를 가진 무한 쿼리를 다시 fetch해야 하는 경우
  - 모든 페이지가 순차적으로 fetch되므로 네트워크 사용량 문제

이럴 때는 `"Limited Infinite Query"`를 사용할 수 있습니다. `maxPages` 옵션을 `getNextPageParam`, `getPreviousPageParam`과 함께 사용하면, 필요할 때 양방향으로 페이지를 가져오면서도 저장 페이지 수를 제한할 수 있습니다.

아래 예시에서는 쿼리 데이터의 `pages` 배열에 3개 페이지만 유지합니다. 다시 fetch가 필요해도 3개 페이지만 순차적으로 다시 fetch됩니다.

```tsx
useInfiniteQuery({
  queryKey: ['projects'],
  queryFn: fetchProjects,
  initialPageParam: 0,
  getNextPageParam: (lastPage, pages) => lastPage.nextCursor,
  getPreviousPageParam: (firstPage, pages) => firstPage.prevCursor,
  maxPages: 3,
})
```

<br>

## 7. What if my API doesn't return a cursor?

API가 cursor를 반환하지 않는다면 `pageParam` 자체를 cursor처럼 사용할 수 있습니다. `getNextPageParam`과 `getPreviousPageParam`은 현재 페이지의 `pageParam`도 함께 받기 때문에, 이를 이용해 다음 또는 이전 페이지의 page param을 계산할 수 있습니다.

```tsx
return useInfiniteQuery({
  queryKey: ['projects'],
  queryFn: fetchProjects,
  initialPageParam: 0,
  getNextPageParam: (lastPage, allPages, lastPageParam) => {
    if (lastPage.length === 0) {
      return undefined
    }
    return lastPageParam + 1
  },
  getPreviousPageParam: (firstPage, allPages, firstPageParam) => {
    if (firstPageParam <= 1) {
      return undefined
    }
    return firstPageParam - 1
  },
})
```

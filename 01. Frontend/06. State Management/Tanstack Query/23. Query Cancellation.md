# Query Cancellation

TanStack Query는 각 query function에 `AbortSignal` 인스턴스를 제공합니다.

- 쿼리가 오래되었거나 비활성 상태가 되면 이 signal은 abort 됩니다.
- 따라서 모든 쿼리는 취소 가능하며, 필요하면 query function 내부에서 취소에 대응할 수 있습니다.
- 이 방식은 일반적인 `async/await` 문법을 유지하면서 자동 취소의 이점을 함께 제공합니다.

`AbortController` API는 대부분의 런타임 환경에서 사용할 수 있습니다.

- 현재 런타임이 이를 지원하지 않으면 polyfill이 필요합니다.

<br>

## 1. Default behavior

기본적으로 다음 경우에도 쿼리는 취소되지 않습니다.

- 컴포넌트가 unmount 된 경우
- Promise가 resolve 되기 전에 더 이상 사용되지 않게 된 경우



이때 Promise가 resolve 되면 결과 데이터는 캐시에 남습니다.

- 쿼리 응답을 받는 도중 컴포넌트가 먼저 unmount 되어도 캐시에 데이터가 저장될 수 있습니다.
- 이후 컴포넌트를 다시 mount 했고, 해당 쿼리가 아직 garbage collection 되지 않았다면 데이터를 그대로 사용할 수 있습니다.



반면 `AbortSignal`을 실제로 소비하면 Promise는 취소됩니다.

- 예: `fetch` abort
- 이 경우 Query도 함께 취소되어야 합니다.
- 쿼리가 취소되면 상태는 이전 상태로 되돌아갑니다.

<br>

## 2. Using fetch

`fetch`를 사용할 때는 `signal`을 요청에 전달하면 됩니다.

- 하나의 `fetch`에 전달할 수 있습니다.
- 여러 `fetch`에 같은 `signal`을 전달할 수도 있습니다.

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: async ({ signal }) => {
    const todosResponse = await fetch('/todos', {
      signal,
    })
    const todos = await todosResponse.json()

    const todoDetails = todos.map(async ({ details }) => {
      const response = await fetch(details, {
        signal,
      })
      return response.json()
    })

    return Promise.all(todoDetails)
  },
})
```

<br>

## 3. Using axios v0.22.0+

`axios` v0.22.0 이상에서는 `signal`을 그대로 전달할 수 있습니다.

```tsx
import axios from 'axios'

const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) =>
    axios.get('/todos', {
      signal,
    }),
})
```

<br>

### 3.1 Using axios with version lower than v0.22.0

`axios` v0.22.0 미만에서는 `CancelToken`을 사용합니다.

- 요청마다 새로운 `CancelToken` source를 생성합니다.
- 요청에는 `cancelToken: source.token`을 전달합니다.
- TanStack Query가 abort 신호를 보내면 `source.cancel(...)`로 요청을 취소합니다.

```tsx
import axios from 'axios'

const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    const CancelToken = axios.CancelToken
    const source = CancelToken.source()

    const promise = axios.get('/todos', {
      cancelToken: source.token,
    })

    signal?.addEventListener('abort', () => {
      source.cancel('Query was cancelled by TanStack Query')
    })

    return promise
  },
})
```

<br>

## 4. Using XMLHttpRequest

`XMLHttpRequest`를 사용할 때도 `signal`에 맞춰 직접 abort 처리를 연결할 수 있습니다.

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    return new Promise((resolve, reject) => {
      var oReq = new XMLHttpRequest()

      oReq.addEventListener('load', () => {
        resolve(JSON.parse(oReq.responseText))
      })

      signal?.addEventListener('abort', () => {
        oReq.abort()
        reject()
      })

      oReq.open('GET', '/todos')
      oReq.send()
    })
  },
})
```

<br>

## 5. Using graphql-request

`graphql-request`에서는 client request method에 `AbortSignal`을 설정할 수 있습니다.

```tsx
const client = new GraphQLClient(endpoint)

const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    client.request({ document: query, signal })
  },
})
```

<br>

### 5.1 Using graphql-request with version lower than v4.0.0

`graphql-request` v4.0.0 미만에서는 `GraphQLClient` 생성자에 `signal`을 설정할 수 있습니다.

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: ({ signal }) => {
    const client = new GraphQLClient(endpoint, {
      signal,
    })

    return client.request(query, variables)
  },
})
```

<br>

## 6. Manual Cancellation

쿼리를 수동으로 취소하고 싶을 때가 있습니다.

- 예: 요청이 오래 걸릴 때 사용자가 직접 취소 버튼을 누르도록 허용하는 경우

이때는 `queryClient.cancelQueries({ queryKey })`를 호출하면 됩니다.

- 해당 쿼리가 취소됩니다.
- 상태는 이전 상태로 되돌아갑니다.
- query function에서 전달된 `signal`을 소비하고 있었다면 Promise도 함께 취소됩니다.

```tsx
const query = useQuery({
  queryKey: ['todos'],
  queryFn: async ({ signal }) => {
    const resp = await fetch('/todos', { signal })
    return resp.json()
  },
})

const queryClient = useQueryClient()

return (
  <button
    onClick={(e) => {
      e.preventDefault()
      queryClient.cancelQueries({ queryKey: ['todos'] })
    }}
  >
    Cancel
  </button>
)
```

<br>

## 7. Cancel Options

Cancel options는 query cancellation 동작을 제어하는 데 사용됩니다.

```tsx
await queryClient.cancelQueries({ queryKey: ['posts'] }, { silent: true })
```

cancel options object는 다음 속성을 지원합니다.

- `silent?: boolean`
  - `true`이면 `CancelledError`가 observer로 전파되지 않습니다.
    - 예: `onError` 콜백
  - 관련 알림도 억제됩니다.
  - reject 대신 retry promise를 반환합니다.
  - 기본값은 `false`입니다.
- `revert?: boolean`
  - `true`이면 in-flight fetch 직전의 query state를 복원합니다.
    - data
    - status
  - `fetchStatus`는 다시 `idle`로 설정됩니다.
  - 이전 데이터가 없었던 경우에만 throw 합니다.
  - 기본값은 `true`입니다.

<br>

## 8. Limitations

Cancellation은 다음 Suspense hook에서는 동작하지 않습니다.

- `useSuspenseQuery`
- `useSuspenseQueries`
- `useSuspenseInfiniteQuery`

# Initial Query Data

쿼리가 필요해지기 전에 캐시에 초기 데이터를 넣는 방법은 여러 가지가 있습니다.

- 선언적 방식
  - 캐시가 비어 있을 때 `initialData`로 쿼리 캐시를 미리 채울 수 있습니다.
- 명령형 방식
  - `queryClient.prefetchQuery`로 데이터를 미리 가져올 수 있습니다.
  - `queryClient.setQueryData`로 데이터를 직접 캐시에 넣을 수 있습니다.

<br>

## 1. Using initialData to prepopulate a query

앱 안에 이미 초기 데이터가 있다면, 그 값을 쿼리에 바로 전달할 수 있습니다.

- 이 경우 `config.initialData`로 초기 데이터를 설정할 수 있습니다.
- 초기 로딩 상태를 건너뛸 수 있습니다.

> IMPORTANT: `initialData`는 캐시에 저장됩니다.  
> 따라서 placeholder, partial, incomplete 데이터에는 적합하지 않습니다.  
> 이런 경우에는 `placeholderData`를 사용해야 합니다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
})
```

<br>

### 1.1 `staleTime` and `initialDataUpdatedAt`

기본적으로 `initialData`는 방금 가져온 데이터처럼 완전히 fresh한 값으로 취급됩니다.

- 그래서 `staleTime` 해석에도 영향을 줍니다.

- `initialData`를 사용하고 `staleTime`을 설정하지 않으면
  - 기본값은 `staleTime: 0`입니다.
  - 마운트 직후 즉시 refetch 됩니다.

```tsx
// initialTodos를 즉시 보여주지만, 마운트 후 바로 todos를 다시 가져옵니다.
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
})
```

- `initialData`와 `staleTime: 1000`을 함께 사용하면, 데이터는 방금 fetch한 것처럼 1000ms 동안 fresh하게 취급됩니다.

```tsx
// initialTodos를 즉시 보여주고, 1000ms 이후 다른 상호작용이 발생하기 전까지는 refetch하지 않습니다.
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
  staleTime: 1000,
})
```

- 그런데 `initialData`가 완전히 fresh하지 않다면 `initialDataUpdatedAt`을 사용할 수 있습니다.
- 이 옵션에는 `initialData`가 마지막으로 업데이트된 시점의 JS timestamp(ms)를 전달합니다.
  - `Date.now()`가 반환하는 값과 같은 형식입니다.
  - Unix timestamp를 사용한다면 `1000`을 곱해 JS timestamp로 변환해야 합니다.


```tsx
// initialTodos를 즉시 보여주고,
// 다른 상호작용이 발생했을 때 stale 여부에 따라 refetch를 결정합니다.
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: initialTodos,
  staleTime: 60 * 1000, // 1 minute
  // 이 값은 10초 전일 수도 있고 10분 전일 수도 있습니다.
  initialDataUpdatedAt: initialTodosUpdatedTimestamp, // eg. 1608412420052
})
```

`initialDataUpdatedAt`을 사용하면 `staleTime`은 원래 목적대로 동작합니다.

- 데이터가 얼마나 fresh해야 하는지 판단할 수 있습니다.
- 동시에 `initialData`가 `staleTime`보다 오래되었다면 마운트 시 refetch할 수도 있습니다.



위 예시에서는 다음을 의미합니다.

- 데이터는 1분 이내로 fresh해야 합니다.
- 쿼리는 `initialData`가 마지막으로 언제 업데이트되었는지 힌트를 받습니다.
- 그 정보를 바탕으로 다시 refetch가 필요한지 스스로 결정합니다.

> 데이터를 prefetched data처럼 다루고 싶다면  
> `prefetchQuery` 또는 `fetchQuery` API로 미리 캐시를 채우는 방식을 권장합니다.  
> 이렇게 하면 `staleTime`을 `initialData`와 독립적으로 설정할 수 있습니다.

<br>

### 1.2 Initial Data Function

초기 데이터를 가져오는 과정이 무겁거나, 매 렌더마다 실행하고 싶지 않다면 `initialData`에 함수를 전달할 수 있습니다.

- 이 함수는 쿼리 초기화 시 한 번만 실행됩니다.
- 메모리와 CPU를 절약할 수 있습니다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: () => fetch('/todos'),
  initialData: () => getExpensiveTodos(),
})
```

<br>

### 1.3 Initial Data from Cache

어떤 경우에는 다른 쿼리의 캐시 결과를 이용해 초기 데이터를 만들 수 있습니다.

대표적인 예시는 다음과 같습니다.

- `todos` 목록 쿼리의 캐시에서 특정 todo를 찾습니다.
- 그 값을 개별 todo 쿼리의 초기 데이터로 사용합니다.

```tsx
const result = useQuery({
  queryKey: ['todo', todoId],
  queryFn: () => fetch('/todos'),
  initialData: () => {
    // 'todos' 쿼리에서 todo 하나를 찾아 이 쿼리의 초기 데이터로 사용합니다.
    return queryClient.getQueryData(['todos'])?.find((d) => d.id === todoId)
  },
})
```

<br>

### 1.4 Initial Data from the cache with `initialDataUpdatedAt`

캐시에서 초기 데이터를 가져온다는 것은, 그 원본 쿼리 데이터가 오래되었을 가능성이 있다는 뜻입니다.

이 경우에는 다음 방식을 권장합니다.

- 즉시 refetch를 막기 위해 인위적인 `staleTime`을 쓰기보다
- 원본 쿼리의 `dataUpdatedAt`을 `initialDataUpdatedAt`에 전달합니다.

이렇게 하면 쿼리 인스턴스는 다음을 스스로 판단할 수 있습니다.

- 현재 데이터가 충분히 fresh한지
- 언제 refetch가 필요한지

```tsx
const result = useQuery({
  queryKey: ['todos', todoId],
  queryFn: () => fetch(`/todos/${todoId}`),
  initialData: () =>
    queryClient.getQueryData(['todos'])?.find((d) => d.id === todoId),
  initialDataUpdatedAt: () =>
    queryClient.getQueryState(['todos'])?.dataUpdatedAt,
})
```

<br>

### 1.5 Conditional Initial Data from Cache

초기 데이터를 가져오는 원본 쿼리가 너무 오래되었다면, 캐시를 아예 사용하지 않고 서버에서 다시 가져오고 싶을 수 있습니다.

이 판단을 쉽게 하려면 `queryClient.getQueryState`를 사용할 수 있습니다.

- 이 메서드는 원본 쿼리에 대한 더 많은 정보를 제공합니다.
- 그중 `state.dataUpdatedAt` timestamp를 이용해 데이터가 충분히 fresh한지 판단할 수 있습니다.

```tsx
const result = useQuery({
  queryKey: ['todo', todoId],
  queryFn: () => fetch(`/todos/${todoId}`),
  initialData: () => {
    // 쿼리 상태를 가져옵니다.
    const state = queryClient.getQueryState(['todos'])

    // 쿼리가 존재하고, 데이터가 10초 이내라면...
    if (state && Date.now() - state.dataUpdatedAt <= 10 * 1000) {
      // 개별 todo를 반환합니다.
      return state.data.find((d) => d.id === todoId)
    }

    // 그렇지 않으면 undefined를 반환하고,
    // 하드 로딩 상태에서 서버 fetch를 진행합니다.
  },
})
```

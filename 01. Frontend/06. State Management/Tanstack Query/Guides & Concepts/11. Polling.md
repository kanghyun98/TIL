# Polling

`refetchInterval`은 query를 타이머 기반으로 다시 가져오게 합니다.

- 밀리초 단위 숫자를 설정하면 됩니다.
- 활성 observer가 하나 이상 있는 동안 query는 N ms마다 실행됩니다.

```tsx
useQuery({
  queryKey: ['prices'],
  queryFn: fetchPrices,
  refetchInterval: 5_000, // every 5 seconds
})
```

Polling은 `staleTime`과 별개입니다.

- query가 fresh 상태여도 정해진 주기에 따라 polling할 수 있습니다.
- `staleTime`이 다른 refetch 동작과 어떻게 상호작용하는지는 별도로 다룹니다.
- `refetchInterval`은 freshness와 관계없이 자체 주기로 실행됩니다.

<br>

## 1. Adapting the Interval to Query State

숫자 대신 함수를 전달하면 현재 query를 기준으로 interval을 계산할 수 있습니다.

- 함수는 `Query` 객체를 받습니다.
- 반환값은 밀리초 단위 숫자입니다.
- polling을 중지하려면 `false`를 반환합니다.

```tsx
useQuery({
  queryKey: ['job', jobId],
  queryFn: () => fetchJobStatus(jobId),
  refetchInterval: (query) => {
    // Stop polling once the job finishes
    if (query.state.data?.status === 'complete') return false
    return 2_000
  },
})
```

- `false`를 반환하면 interval 타이머가 제거됩니다.
- query 결과가 바뀌어 함수가 다시 양수를 반환하면 polling은 자동으로 재개됩니다.

<br>

## 2. Background Polling

기본적으로 polling은 브라우저 탭이 포커스를 잃으면 일시 중지됩니다.

- 사용자가 다른 탭에 있어도 데이터를 계속 최신 상태로 유지해야 하는 인터페이스에서는
- 이 동작을 비활성화할 수 있습니다.

```tsx
useQuery({
  queryKey: ['portfolio'],
  queryFn: fetchPortfolio,
  refetchInterval: 30_000,
  refetchIntervalInBackground: true,
})
```

<br>

## 3. Pausing Polling

`refetchInterval`에 함수를 전달하고 컴포넌트 상태를 클로저로 참조하면 polling 실행 시점을 제어할 수 있습니다.

```tsx
useQuery({
  queryKey: ['prices', tokenAddress],
  queryFn: () => fetchPrice(tokenAddress),
  refetchInterval: () => {
    if (!tokenAddress || isPaused) return false
    return 15_000
  },
})
```

<br>

## 4. Polling with Offline Support

TanStack Query는 브라우저의 online/offline 이벤트를 감지해 연결 상태를 판단합니다.

- Electron
- 일부 임베디드 WebView

이처럼 해당 이벤트가 안정적으로 발생하지 않는 환경에서는 `networkMode: 'always'`를 설정해 연결성 체크를 건너뜁니다.

```tsx
useQuery({
  queryKey: ['chainStatus'],
  queryFn: fetchChainStatus,
  refetchInterval: 10_000,
  networkMode: 'always',
})
```

네트워크 모드에 대한 내용은 별도로 다룹니다.

<br>

## 5. Note on Deduplication

각 `QueryObserver`는 자체 타이머를 실행합니다.

- `refetchInterval`을 사용하는 `useQuery` 컴포넌트마다 개별 타이머가 동작합니다.
- 같은 key를 구독하는 두 컴포넌트가 각각 `refetchInterval: 5000`을 사용하면
  - 두 타이머는 각각 5초마다 실행됩니다.

중복 제거가 적용되는 대상은 동시에 진행 중인 fetch입니다.

- 두 타이머가 같은 시점에 실행되면
  - 실제 네트워크 요청은 하나만 발생합니다.

정리하면 다음과 같습니다.

- 타이머는 observer 단위입니다.
- deduplication은 query 단위입니다.

<br>

## 6. Non-browser Environments

React Native 같은 non-browser 런타임에서는 기본 online/offline 이벤트와 focus 이벤트를 사용할 수 없습니다.

- `focusManager`
- `onlineManager`

이들을 네이티브 앱 상태 API에 연결하는 방법은 별도로 다룹니다.

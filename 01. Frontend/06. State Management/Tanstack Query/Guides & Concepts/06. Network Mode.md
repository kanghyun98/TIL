# Network Mode

TanStack Query는 네트워크 연결이 없을 때 `Queries`와 `Mutations`가 어떻게 동작할지 구분하기 위해 세 가지 네트워크 모드를 제공합니다.

이 모드는 다음 방식으로 설정할 수 있습니다.

- 각 Query / Mutation 별 개별 설정
- query / mutation defaults를 통한 전역 설정

TanStack Query는 주로 데이터 페칭 라이브러리와 함께 사용되므로, 기본 네트워크 모드는 `online`입니다.

<br>

## 1. Network Mode: online

이 모드에서는 네트워크 연결이 있어야 `Queries`와 `Mutations`가 실행됩니다.

- 기본 모드입니다.
- Query의 fetch가 시작된 뒤, 네트워크 연결이 없어 요청을 보낼 수 없는 경우
  - Query는 현재 상태를 그대로 유지합니다.
  - 상태는 `pending`, `error`, `success` 중 하나입니다.
- 대신 추가로 `fetchStatus`가 제공됩니다.

`fetchStatus`는 다음 값 중 하나입니다.

- `fetching`
  - `queryFn`이 실제로 실행 중인 상태
  - 요청이 전송 중인 상태
- `paused`
  - Query가 실행되지 않는 상태
  - 네트워크가 다시 연결될 때까지 일시 정지된 상태
- `idle`
  - 현재 fetch 중도 아니고, paused 상태도 아님

편의를 위해 다음 플래그도 파생되어 제공됩니다.

- `isFetching`
- `isPaused`

> 로딩 스피너를 표시할 때 `pending` 상태만 확인하는 것으로는 충분하지 않을 수 있습니다. Query는 처음 마운트될 때 네트워크 연결이 없으면 `state: 'pending'` 이면서 `fetchStatus: 'paused'`일 수 있습니다.

Query가 온라인 상태에서 실행되다가 fetch 도중 오프라인이 되면, TanStack Query는 retry 메커니즘도 일시 정지합니다.

- paused된 Query는 네트워크가 다시 연결되면 이어서 실행됩니다.
- 이는 `refetchOnReconnect`와는 별개입니다.
  - 이 경우는 refetch가 아니라 continue입니다.
- 그 사이 Query가 `cancelled`되었다면 다시 이어서 실행되지 않습니다.

<br>

## 2. Network Mode: always

이 모드에서는 TanStack Query가 항상 fetch를 시도하며, 온라인 / 오프라인 상태를 무시합니다.

다음과 같은 환경에서 적합합니다.

- Query가 동작하는 데 활성 네트워크 연결이 필요 없는 경우
- 예를 들어
  - `AsyncStorage`에서만 데이터를 읽는 경우
  - `queryFn`이 단순히 `Promise.resolve(5)`를 반환하는 경우

이 모드의 동작은 다음과 같습니다.

- 네트워크 연결이 없다는 이유로 Query가 paused되지 않습니다.
- retry도 paused되지 않습니다.
  - 실패하면 Query는 `error` 상태가 됩니다.
- 이 모드에서는 `refetchOnReconnect`의 기본값이 `false`입니다.
  - 네트워크 재연결이 stale Query를 다시 가져와야 한다는 좋은 신호가 아니기 때문입니다.
  - 필요하면 직접 켤 수 있습니다.

<br>

## 3. Network Mode: offlineFirst

이 모드는 앞의 두 옵션 사이의 중간 지점입니다.

동작 방식은 다음과 같습니다.

- TanStack Query는 `queryFn`을 한 번 실행합니다.
- 이후 retry는 paused합니다.

이 모드는 다음과 같은 경우에 유용합니다.

- `serviceWorker`가 요청을 가로채 캐싱하는 경우
- HTTP 캐싱에서 `Cache-Control` 헤더를 사용하는 경우

이런 상황에서는 첫 번째 fetch가 오프라인 저장소나 캐시에서 응답되어 성공할 수 있습니다.

하지만 다음 경우도 있습니다.

- 캐시 미스가 발생함
- 네트워크 요청이 실제로 나감
- 그 요청이 실패함

이 경우 이 모드는 `online` Query처럼 동작하며 retry를 paused합니다.

<br>

## 4. Devtools

TanStack Query Devtools는 fetch되어야 하지만 네트워크 연결이 없어 실행되지 못하는 Query를 paused 상태로 표시합니다.

또한 다음 기능도 제공합니다.

- `Mock offline behavior` 토글 버튼

이 버튼은 실제 네트워크 연결을 변경하지는 않습니다.

- 실제 네트워크 연결 변경은 브라우저 Devtools에서 할 수 있습니다.
- 이 버튼은 `OnlineManager`를 offline 상태로 설정합니다.

<br>

## 5. Signature

- `networkMode: 'online' | 'always' | 'offlineFirst'`
  - optional
  - 기본값은 `'online'`

# Scroll Restoration

<br>

## 1. Scroll Restoration

브라우저에서 이전에 방문했던 페이지로 이동하면, 일반적으로 이전에 머물렀던 스크롤 위치로 돌아갑니다. 이를 **scroll restoration**이라고 합니다.

하지만 웹 애플리케이션이 클라이언트 사이드 데이터 페칭 방식으로 이동하면서, scroll restoration은 제대로 동작하지 않는 경우가 생겼습니다. TanStack Query를 사용하면 이 문제가 줄어듭니다.

<br>

TanStack Query는 scroll restoration을 직접 구현하지 않습니다. 대신 SPA에서 scroll restoration이 깨지는 주요 원인 중 하나인 **refetch로 인한 UI reset**을 줄입니다.

TanStack Query는 이전에 가져온 데이터를 캐시에 유지합니다. 필요한 경우 `placeholderData`를 사용할 수도 있습니다. 그 결과 이전 페이지로 돌아갔을 때 다음 흐름이 가능해집니다.

- 페이지가 즉시 렌더링됨
- 레이아웃이 안정적으로 유지됨
- 라우터가 처리하는 scroll restoration이 더 안정적으로 동작함

<br>

scroll restoration은 라우터에서 처리할 수 있습니다.

- React Router의 `ScrollRestoration`
- TanStack Router의 scroll restoration
- history 기반의 작은 custom solution

<br>

TanStack Query에서는 기본적으로 모든 query에 대해 scroll restoration이 동작합니다.

여기에는 다음 query도 포함됩니다.

- paginated query
- infinite query

그 이유는 query 결과가 캐시되며, query가 렌더링될 때 동기적으로 가져올 수 있기 때문입니다.

따라서 다음 조건을 만족하면 scroll restoration은 기본적으로 동작합니다.

- query가 충분히 오래 캐시되어 있음
- query가 garbage collected 되지 않음

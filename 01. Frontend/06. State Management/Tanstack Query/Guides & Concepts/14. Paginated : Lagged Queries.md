# Paginated / Lagged Queries

페이지네이션 데이터를 렌더링하는 것은 매우 일반적인 UI 패턴입니다.  
TanStack Query에서는 query key에 페이지 정보를 포함하면 이 패턴이 그대로 동작합니다.

```tsx
const result = useQuery({
  queryKey: ['projects', page],
  queryFn: () => fetchProjects(page),
})
```

<br>

하지만 이 단순한 예제를 실행해 보면 한 가지 이상한 점이 보입니다.

- 새로운 페이지마다 완전히 새로운 query로 처리됩니다.
- 그래서 UI가 success 상태와 pending 상태 사이를 반복해서 오갑니다.

이 경험은 최적이 아닙니다. TanStack Query는 `placeholderData` 기능으로 이 문제를 해결할 수 있습니다.

<br>

## 1. Better Paginated Queries with `placeholderData`

다음처럼 query의 `pageIndex` 또는 `cursor`를 증가시키는 상황을 생각해 볼 수 있습니다.

`useQuery`만으로도 기술적으로는 동작합니다. 하지만 페이지나 커서가 바뀔 때마다 각 query가 새로 생성되고 제거되기 때문에, UI는 success 상태와 pending 상태 사이를 반복해서 오가게 됩니다.

`placeholderData`를 `(previousData) => previousData` 또는 `keepPreviousData`로 설정하면 다음과 같은 점이 달라집니다.

- query key가 변경되어도, 새 데이터를 요청하는 동안 마지막 성공 데이터가 유지됩니다.
- 새 데이터가 도착하면 이전 데이터가 자연스럽게 교체됩니다.
- 현재 query가 어떤 데이터를 제공 중인지 `isPlaceholderData`로 알 수 있습니다.

```tsx
import { keepPreviousData, useQuery } from '@tanstack/react-query'
import React from 'react'

function Todos() {
  const [page, setPage] = React.useState(0)

  const fetchProjects = (page = 0) =>
    fetch('/api/projects?page=' + page).then((res) => res.json())

  const { isPending, isError, error, data, isFetching, isPlaceholderData } =
    useQuery({
      queryKey: ['projects', page],
      queryFn: () => fetchProjects(page),
      placeholderData: keepPreviousData,
    })

  return (
    <div>
      {isPending ? (
        <div>Loading...</div>
      ) : isError ? (
        <div>Error: {error.message}</div>
      ) : (
        <div>
          {data.projects.map((project) => (
            <p key={project.id}>{project.name}</p>
          ))}
        </div>
      )}

      <span>Current Page: {page + 1}</span>

      <button
        onClick={() => setPage((old) => Math.max(old - 1, 0))}
        disabled={page === 0}
      >
        Previous Page
      </button>

      <button
        onClick={() => {
          if (!isPlaceholderData && data.hasMore) {
            setPage((old) => old + 1)
          }
        }}
        disabled={isPlaceholderData || !data?.hasMore}
      >
        Next Page
      </button>

      {isFetching ? <span> Loading...</span> : null}
    </div>
  )
}
```

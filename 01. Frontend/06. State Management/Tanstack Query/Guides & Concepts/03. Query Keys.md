# Query Keys

TanStack Query는 query key를 기반으로 쿼리 캐싱을 관리합니다.

query key는 최상위 레벨에서 반드시 배열이어야 합니다. 하나의 문자열만 담은 단순한 배열일 수도 있고, 여러 문자열과 중첩 객체를 포함한 복잡한 배열일 수도 있습니다. query key가 `JSON.stringify`로 직렬화 가능하고, 해당 쿼리의 데이터에 대해 고유하다면 사용할 수 있습니다.

<br>

## 1. Simple Query Keys

가장 단순한 key는 상수 값을 담은 배열입니다.

이 형식은 다음과 같은 경우에 적합합니다.

- 일반적인 리스트 또는 인덱스 리소스
- 계층 구조가 없는 리소스

```tsx
// A list of todos
useQuery({ queryKey: ['todos'], ... })

// Something else, whatever!
useQuery({ queryKey: ['something', 'special'], ... })
```

<br>

## 2. Array Keys with variables

쿼리가 자신의 데이터를 더 구체적으로 설명해야 한다면, 문자열과 함께 직렬화 가능한 객체를 포함한 배열을 사용할 수 있습니다.

이 형식은 다음과 같은 경우에 적합합니다.

- 계층적이거나 중첩된 리소스
  - 항목을 고유하게 식별하기 위해 ID, 인덱스 또는 다른 원시값을 전달하는 것이 일반적입니다.
- 추가 파라미터가 있는 쿼리
  - 추가 옵션 객체를 전달하는 것이 일반적입니다.

```tsx
// An individual todo
useQuery({ queryKey: ['todo', 5], ... })

// An individual todo in a "preview" format
useQuery({ queryKey: ['todo', 5, { preview: true }], ... })

// A list of todos that are "done"
useQuery({ queryKey: ['todos', { type: 'done' }], ... })
```

<br>

## 3. Query Keys are hashed deterministically!

객체 내부의 key 순서와 관계없이, 아래 쿼리들은 모두 동일한 것으로 간주됩니다.

```tsx
useQuery({ queryKey: ['todos', { status, page }], ... })
useQuery({ queryKey: ['todos', { page, status }], ... })
useQuery({ queryKey: ['todos', { page, status, other: undefined }], ... })
```

반면 아래 query key들은 서로 동일하지 않습니다. 배열 항목의 순서는 중요합니다.

```tsx
useQuery({ queryKey: ['todos', status, page], ... })
useQuery({ queryKey: ['todos', page, status], ... })
useQuery({ queryKey: ['todos', undefined, page, status], ... })
```

<br>

## 4. If your query function depends on a variable, include it in your query key

query key는 가져오는 데이터를 고유하게 설명하므로, query function에서 사용하는 값 중 변경될 수 있는 변수는 query key에 포함되어야 합니다.

```tsx
function Todos({ todoId }) {
  const result = useQuery({
    queryKey: ['todos', todoId],
    queryFn: () => fetchTodoById(todoId),
  })
}
```

query key는 query function의 의존성 역할도 합니다. 의존하는 변수를 query key에 추가하면, 쿼리는 서로 독립적으로 캐시되고 변수가 바뀔 때마다 자동으로 다시 가져옵니다. 이는 `staleTime` 설정에 따라 달라집니다.

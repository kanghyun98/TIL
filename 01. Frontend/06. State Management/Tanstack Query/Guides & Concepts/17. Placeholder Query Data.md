# Placeholder Query Data

## 1. What is placeholder data?

placeholder data는 쿼리가 이미 데이터를 가진 것처럼 동작하게 합니다.

- `initialData`와 비슷하지만, 데이터는 캐시에 영속적으로 저장되지 않습니다.
- 실제 데이터를 백그라운드에서 가져오는 동안, 부분 데이터나 가짜 데이터로 UI를 먼저 렌더링할 때 유용합니다.

  - 예를 들어, 개별 블로그 글 쿼리는 부모 블로그 글 목록에서 `title`과 본문 일부만 포함된 preview 데이터를 가져올 수 있습니다.

  - 이런 부분 데이터는 개별 쿼리 결과에 그대로 저장하고 싶지는 않을 수 있습니다.

  - 하지만 실제 전체 데이터를 가져오는 동안 콘텐츠 레이아웃을 빠르게 보여주는 데는 유용합니다.




쿼리에 placeholder data를 미리 제공하는 방법은 몇 가지가 있습니다.

- Declaratively:
  - 쿼리에 `placeholderData`를 제공해, 캐시가 비어 있을 때 미리 채웁니다.
- Imperatively:
  - `queryClient`로 데이터를 prefetch 또는 fetch하고 `placeholderData` 옵션을 사용합니다.



`placeholderData`를 사용하면 Query는 `pending` 상태로 시작하지 않습니다.

- 표시할 데이터가 있기 때문에 `success` 상태로 시작합니다.
- 이 데이터가 실제 데이터가 아니라 placeholder data라는 점은 `isPlaceholderData` 플래그가 `true`로 설정되어 구분됩니다.

<br>

## 2. Placeholder Data as a Value

`placeholderData`에 값을 직접 넣어 사용할 수 있습니다.

```tsx
function Todos() {
  const result = useQuery({
    queryKey: ['todos'],
    queryFn: () => fetch('/todos'),
    placeholderData: placeholderTodos,
  })
}
```

<br>

### 2.1. Placeholder Data Memoization

쿼리의 placeholder data를 가져오는 과정이 무겁거나, 매 렌더마다 수행하고 싶지 않다면 값을 memoize할 수 있습니다.

```tsx
function Todos() {
  const placeholderData = useMemo(() => generateFakeTodos(), [])

  const result = useQuery({
    queryKey: ['todos'],
    queryFn: () => fetch('/todos'),
    placeholderData,
  })
}
```

<br>

## 3. Placeholder Data as a Function

`placeholderData`는 함수로도 사용할 수 있습니다.

- 이 경우 이전에 성공한 Query의 데이터와 Query 메타 정보에 접근할 수 있습니다.
- 한 쿼리의 데이터를 다른 쿼리의 placeholder data로 사용할 때 유용합니다.

`QueryKey`가 바뀔 때도 활용할 수 있습니다.

- 예를 들어 `['todos', 1]`에서 `['todos', 2]`로 바뀌는 경우,
- 다음 쿼리로 전환되는 동안 로딩 스피너를 보여주는 대신 기존 데이터를 계속 표시할 수 있습니다.

자세한 내용은 `Paginated Queries`를 참고하면 됩니다.

```tsx
const result = useQuery({
  queryKey: ['todos', id],
  queryFn: () => fetch(`/todos/${id}`),
  placeholderData: (previousData, previousQuery) => previousData,
})
```

<br>

### 3.1. Placeholder Data from Cache

경우에 따라 다른 쿼리의 캐시 결과를 현재 쿼리의 placeholder data로 사용할 수 있습니다.

좋은 예시는 블로그 글 목록 쿼리의 캐시 데이터에서 글 preview 버전을 찾아, 개별 글 쿼리의 placeholder data로 사용하는 경우입니다.

```tsx
function BlogPost({ blogPostId }) {
  const queryClient = useQueryClient()

  const result = useQuery({
    queryKey: ['blogPost', blogPostId],
    queryFn: () => fetch(`/blogPosts/${blogPostId}`),
    placeholderData: () => {
      // 'blogPosts' 쿼리의 더 작은 preview 버전을
      // 현재 blogPost 쿼리의 placeholder data로 사용
      return queryClient
        .getQueryData(['blogPosts'])
        ?.find((d) => d.id === blogPostId)
    },
  })
}
```

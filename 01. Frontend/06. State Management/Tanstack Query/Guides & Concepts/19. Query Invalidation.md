# Query Invalidation

쿼리가 stale 상태가 되기를 기다렸다가 다시 가져오는 방식은 항상 충분하지 않습니다.

특히 사용자의 어떤 동작으로 인해 쿼리 데이터가 오래되었다는 것을 확실히 아는 경우에는 더 직접적인 처리가 필요합니다.

이때 `QueryClient`의 `invalidateQueries`를 사용하면 쿼리를 stale 상태로 표시하고, 필요하면 다시 가져오도록 만들 수 있습니다.

```tsx
// 캐시에 있는 모든 쿼리 무효화
queryClient.invalidateQueries()

// `todos`로 시작하는 키를 가진 모든 쿼리 무효화
queryClient.invalidateQueries({ queryKey: ['todos'] })
```

> 다른 라이브러리는 정규화된 캐시를 기준으로 새 데이터를 로컬 쿼리에 직접 반영하거나, 스키마를 추론해 갱신하려고 시도할 수 있습니다. TanStack Query는 정규화된 캐시를 직접 관리하는 수작업 대신, 대상 지정 무효화와 백그라운드 리패칭, 그리고 원자적 업데이트를 권장합니다.

`invalidateQueries`로 쿼리를 무효화하면 다음 두 가지가 일어납니다.

- 해당 쿼리가 stale 상태로 표시됩니다.
  - 이 stale 상태는 `useQuery` 또는 관련 훅에 설정한 `staleTime`보다 우선합니다.
- 해당 쿼리가 현재 `useQuery` 또는 관련 훅을 통해 렌더링 중이라면 백그라운드에서 다시 가져옵니다.

<br>

## 1. Query Matching with invalidateQueries

`invalidateQueries`, `removeQueries`처럼 partial query matching을 지원하는 API를 사용하면 다음과 같이 매칭할 수 있습니다.

- prefix 기준으로 여러 쿼리 매칭
- exact query 기준으로 정확하게 매칭

사용할 수 있는 필터 타입은 Query Filters를 참고하면 됩니다.

다음 예제에서는 `todos` prefix를 사용해 query key가 `todos`로 시작하는 모든 쿼리를 무효화합니다.

```tsx
import { useQuery, useQueryClient } from '@tanstack/react-query'

// context에서 QueryClient 가져오기
const queryClient = useQueryClient()

queryClient.invalidateQueries({ queryKey: ['todos'] })

// 아래 두 쿼리는 모두 무효화됩니다.
const todoListQuery = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})

const todoListQuery = useQuery({
  queryKey: ['todos', { page: 1 }],
  queryFn: fetchTodoList,
})
```



더 구체적인 query key를 `invalidateQueries`에 넘기면, 특정 변수까지 포함해 무효화할 수도 있습니다.

```tsx
queryClient.invalidateQueries({
  queryKey: ['todos', { type: 'done' }],
})

// 아래 쿼리는 무효화됩니다.
const todoListQuery = useQuery({
  queryKey: ['todos', { type: 'done' }],
  queryFn: fetchTodoList,
})

// 하지만 아래 쿼리는 무효화되지 않습니다.
const todoListQuery = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})
```



`invalidateQueries`는 매우 유연합니다. `todos` 쿼리 중에서도 추가 변수나 하위 키가 없는 쿼리만 무효화하고 싶다면 `exact: true` 옵션을 사용할 수 있습니다.

```tsx
queryClient.invalidateQueries({
  queryKey: ['todos'],
  exact: true,
})

// 아래 쿼리는 무효화됩니다.
const todoListQuery = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
})

// 하지만 아래 쿼리는 무효화되지 않습니다.
const todoListQuery = useQuery({
  queryKey: ['todos', { type: 'done' }],
  queryFn: fetchTodoList,
})
```



더 세밀한 제어가 필요하다면 `predicate` 함수를 `invalidateQueries`에 전달할 수 있습니다.

이 함수는 query cache의 각 `Query` 인스턴스를 받아, 해당 쿼리를 무효화할지 여부를 `true` 또는 `false`로 반환합니다.

```tsx
queryClient.invalidateQueries({
  predicate: (query) =>
    query.queryKey[0] === 'todos' && query.queryKey[1]?.version >= 10,
})

// 아래 쿼리는 무효화됩니다.
const todoListQuery = useQuery({
  queryKey: ['todos', { version: 20 }],
  queryFn: fetchTodoList,
})

// 아래 쿼리는 무효화됩니다.
const todoListQuery = useQuery({
  queryKey: ['todos', { version: 10 }],
  queryFn: fetchTodoList,
})

// 하지만 아래 쿼리는 무효화되지 않습니다.
const todoListQuery = useQuery({
  queryKey: ['todos', { version: 5 }],
  queryFn: fetchTodoList,
})
```

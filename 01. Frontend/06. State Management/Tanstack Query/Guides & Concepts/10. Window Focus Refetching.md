# Window Focus Refetching

사용자가 애플리케이션을 떠났다가 다시 돌아왔을 때, 쿼리 데이터가 stale 상태라면 TanStack Query는 백그라운드에서 자동으로 최신 데이터를 다시 요청합니다.

이 동작은 `refetchOnWindowFocus` 옵션으로 전역 또는 쿼리 단위에서 비활성화할 수 있습니다.

#### 1.1 Disabling Globally

전역에서 비활성화하려면 `QueryClient`의 기본 옵션에 `refetchOnWindowFocus: false`를 설정합니다.

```tsx
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      refetchOnWindowFocus: false, // default: true
    },
  },
})

function App() {
  return <QueryClientProvider client={queryClient}>...</QueryClientProvider>
}
```

<br>

#### 1.2 Disabling Per-Query

특정 쿼리만 비활성화하려면 해당 `useQuery`에 `refetchOnWindowFocus: false`를 설정합니다.

```tsx
useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodos,
  refetchOnWindowFocus: false,
})
```

<br>

## 2. Custom Window Focus Event

드문 경우, TanStack Query가 재검증을 수행할 때 사용할 윈도우 포커스 이벤트를 직접 관리하고 싶을 수 있습니다.

이를 위해 TanStack Query는 `focusManager.setEventListener`를 제공합니다.

- 이 함수는 윈도우가 포커스되었을 때 실행해야 하는 콜백을 전달합니다.
- 이 콜백을 사용해 원하는 이벤트를 직접 연결할 수 있습니다.
- `focusManager.setEventListener`를 호출하면, 이전에 설정된 핸들러는 제거됩니다.
- 대부분의 경우 제거되는 핸들러는 기본 핸들러입니다.
- 이후에는 새로 설정한 핸들러가 사용됩니다.

아래는 기본 핸들러입니다.

```tsx
focusManager.setEventListener((handleFocus) => {
  // Listen to visibilitychange
  if (typeof window !== 'undefined' && window.addEventListener) {
    const visibilitychangeHandler = () => {
      handleFocus(document.visibilityState === 'visible')
    }
    window.addEventListener('visibilitychange', visibilitychangeHandler, false)
    return () => {
      // Be sure to unsubscribe if a new handler is set
      window.removeEventListener('visibilitychange', visibilitychangeHandler)
    }
  }
})
```

## 3. Managing Focus in React Native

React Native는 `window` 이벤트 리스너 대신 `AppState` 모듈을 통해 포커스 정보를 제공합니다.

앱 상태가 `"active"`로 바뀔 때 업데이트를 트리거하려면 `AppState`의 `"change"` 이벤트를 사용할 수 있습니다.

```tsx
import { AppState } from 'react-native'
import { focusManager } from '@tanstack/react-query'

function onAppStateChange(status: AppStateStatus) {
  if (Platform.OS !== 'web') {
    focusManager.setFocused(status === 'active')
  }
}

useEffect(() => {
  const subscription = AppState.addEventListener('change', onAppStateChange)

  return () => subscription.remove()
}, [])
```

<br>

## 4. Managing Focus State

포커스 상태를 직접 설정할 수도 있습니다.

- 기본 포커스 상태를 직접 덮어쓸 수 있습니다.
- `undefined`를 전달하면 기본 포커스 확인 방식으로 되돌릴 수 있습니다.

```tsx
import { focusManager } from '@tanstack/react-query'

// Override the default focus state
focusManager.setFocused(true)

// Fallback to the default focus check
focusManager.setFocused(undefined)
```

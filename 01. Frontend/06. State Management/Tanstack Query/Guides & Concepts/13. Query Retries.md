# Query Retries

`useQuery` 쿼리가 실패하면 (query function이 error를 throw하면), TanStack Query는 다음 조건에서 자동으로 재시도합니다.

- 해당 요청의 연속 재시도 횟수가 최대값에 도달하지 않은 경우
  - 기본값은 `3`입니다.
- 재시도 허용 여부를 결정하는 함수가 제공된 경우

<br>

재시도는 전역 및 개별 쿼리에서 설정할 수 있습니다.

- `retry = false`: 재시도를 비활성화합니다.
- `retry = 6`: 최종 에러를 표시하기 전까지 `6`번 재시도합니다.
- `retry = true`: 실패한 요청을 무한히 재시도합니다.
- `retry = (failureCount, error) => ...`
  - 실패 원인에 따라 사용자 정의 로직으로 재시도 여부를 결정합니다.
  - `failureCount`는 첫 번째 재시도 시점에 `0`부터 시작합니다.

> 서버에서는 서버 렌더링을 최대한 빠르게 하기 위해 재시도 기본값이 `0`입니다.

```tsx
import { useQuery } from '@tanstack/react-query'

// 특정 쿼리의 재시도 횟수 설정
const result = useQuery({
  queryKey: ['todos', 1],
  queryFn: fetchTodoListPage,
  retry: 10, // 에러를 표시하기 전까지 실패한 요청을 10번 재시도
})
```

> `useQuery`의 `error` 프로퍼티 내용은 마지막 재시도 전까지 `failureReason` 응답 프로퍼티에 포함됩니다.
>
> 위 예시에서는 전체 `10`번 시도 중 처음 `9`번의 재시도 동안 에러 내용이 `failureReason`에 포함됩니다. 모든 재시도 이후에도 에러가 계속되면, 마지막 시도 후에는 `error`에 포함됩니다.

<br>

## 1. Retry Delay

기본적으로 TanStack Query의 재시도는 요청이 실패한 직후 바로 실행되지 않습니다.

- 일반적인 방식처럼, 각 재시도 시도에는 점진적으로 증가하는 백오프 딜레이가 적용됩니다.

기본 `retryDelay`는 시도할 때마다 `2`배로 증가하며, 시작값은 `1000ms`이고 최대 `30초`를 넘지 않습니다.

```tsx
// 모든 쿼리에 적용
import {
  QueryCache,
  QueryClient,
  QueryClientProvider,
} from '@tanstack/react-query'

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      retryDelay: (attemptIndex) => Math.min(1000 * 2 ** attemptIndex, 30000),
    },
  },
})

function App() {
  return <QueryClientProvider client={queryClient}>...</QueryClientProvider>
}
```

권장되지는 않지만, `retryDelay` 함수나 정수 값은 Provider와 개별 쿼리 옵션 모두에서 오버라이드할 수 있습니다.

- 함수 대신 정수로 설정하면, 재시도 횟수와 관계없이 항상 같은 시간만큼 대기합니다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodoList,
  retryDelay: 1000, // 재시도 횟수와 관계없이 항상 1000ms 대기
})
```

## 2. Background Retry Behavior

`refetchInterval`과 `refetchIntervalInBackground: true`를 함께 사용하면, 브라우저 탭이 비활성화된 동안 재시도가 일시 중지됩니다.

- 재시도는 일반 refetch와 동일한 포커스 동작을 따르기 때문입니다.

백그라운드에서도 재시도를 계속해야 한다면, 재시도를 비활성화하고 사용자 정의 refetch 전략을 구현하는 방식을 고려할 수 있습니다.

```tsx
const result = useQuery({
  queryKey: ['todos'],
  queryFn: fetchTodos,
  refetchInterval: (query) => {
    // 에러 상태일 때 더 자주 refetch
    return query.state.status === 'error' ? 5000 : 30000
  },
  refetchIntervalInBackground: true,
  retry: false, // 내장 재시도 비활성화
})
```

이 방식은 백그라운드 refetch를 유지하면서 재시도 시점을 직접 제어할 수 있게 해줍니다.

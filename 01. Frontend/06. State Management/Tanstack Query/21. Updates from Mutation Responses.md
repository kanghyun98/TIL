# Updates from Mutation Responses

서버에서 객체를 업데이트하는 mutation은 응답으로 새로운 객체를 자동으로 반환하는 경우가 많습니다.

- 이미 가진 데이터를 다시 가져오기 위해 해당 item의 query를 refetch하면 네트워크 호출이 낭비됩니다.
- 이럴 때는 mutation 함수가 반환한 객체를 활용해 `QueryClient`의 `setQueryData`로 기존 query 데이터를 즉시 업데이트할 수 있습니다.

```tsx
const queryClient = useQueryClient()

const mutation = useMutation({
  mutationFn: editTodo,
  onSuccess: (data) => {
    queryClient.setQueryData(['todo', { id: 5 }], data)
  },
})

mutation.mutate({
  id: 5,
  name: 'Do the laundry',
})

// The query below will be updated with the response from the
// successful mutation
const { status, data, error } = useQuery({
  queryKey: ['todo', { id: 5 }],
  queryFn: fetchTodoById,
})
```

<br>

이 `onSuccess` 로직을 재사용 가능한 mutation에 묶고 싶다면, 다음처럼 custom hook으로 만들 수 있습니다.

```tsx
const useMutateTodo = () => {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: editTodo,
    // Notice the second argument is the variables object that the `mutate` function receives
    onSuccess: (data, variables) => {
      queryClient.setQueryData(['todo', { id: variables.id }], data)
    },
  })
}
```

<br>

## 1. Immutability

`setQueryData`로 업데이트할 때는 반드시 immutable 방식으로 처리해야 합니다.

- 캐시에서 가져온 데이터를 직접 변경하지 마세요.
- 처음에는 동작하는 것처럼 보여도, 이후 미묘한 버그로 이어질 수 있습니다.

```tsx
queryClient.setQueryData(['posts', { id }], (oldData) => {
  if (oldData) {
    // ❌ do not try this
    oldData.title = 'my new post title'
  }
  return oldData
})

queryClient.setQueryData(
  ['posts', { id }],
  // ✅ this is the way
  (oldData) =>
    oldData
      ? {
          ...oldData,
          title: 'my new post title',
        }
      : oldData,
)
```

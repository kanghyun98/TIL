# Disabling/Pausing Queries

쿼리가 자동으로 실행되지 않게 하려면 `enabled = false` 옵션을 사용할 수 있습니다.  
`enabled`는 boolean을 반환하는 콜백도 받을 수 있습니다.

`enabled`가 `false`이면:

- 쿼리에 캐시된 데이터가 있으면 `status === 'success'` 또는 `isSuccess` 상태로 초기화됩니다.
- 쿼리에 캐시된 데이터가 없으면 `status === 'pending'` 및 `fetchStatus === 'idle'` 상태로 시작합니다.
- 마운트 시 자동으로 fetch되지 않습니다.
- 백그라운드에서 자동으로 refetch되지 않습니다.
- 원래 refetch를 유발하던 query client의 `invalidateQueries`, `refetchQueries` 호출을 무시합니다.
- `useQuery`가 반환하는 `refetch`로 수동 fetch를 트리거할 수 있습니다. 다만 `skipToken`에서는 동작하지 않습니다.

> TypeScript 사용자는 `enabled = false`의 대안으로 `skipToken`을 선호할 수 있습니다.

```tsx
function Todos() {
  const { isLoading, isError, data, error, refetch, isFetching } = useQuery({
    queryKey: ['todos'],
    queryFn: fetchTodoList,
    enabled: false,
  })

  return (
    <div>
      <button onClick={() => refetch()}>Fetch Todos</button>
      {data ? (
        <ul>
          {data.map((todo) => (
            <li key={todo.id}>{todo.title}</li>
          ))}
        </ul>
      ) : isError ? (
        <span>Error: {error.message}</span>
      ) : isLoading ? (
        <span>Loading...</span>
      ) : (
        <span>Not ready ...</span>
      )}

      <div>{isFetching ? 'Fetching...' : null}</div>
    </div>
  )
}
```

쿼리를 영구적으로 비활성화하면 TanStack Query가 제공하는 여러 장점(예: 백그라운드 refetch)을 활용하지 않게 되며, 권장되는 방식도 아닙니다.  
이 방식은 쿼리가 언제 실행되어야 하는지 의존성을 선언하는 선언형 접근에서, 클릭할 때마다 fetch하는 명령형 방식으로 바뀝니다.  
또한 `refetch`에 파라미터를 전달할 수도 없습니다.  
대부분의 경우, 필요한 것은 초기 fetch만 지연하는 lazy query입니다.

<br>

## 1. Lazy Queries

`enabled`는 쿼리를 영구적으로 비활성화할 때뿐 아니라, 나중에 활성화하거나 비활성화할 때도 사용할 수 있습니다.  
좋은 예시는 filter form입니다. 사용자가 filter 값을 입력한 뒤에만 첫 요청을 보내고 싶을 수 있습니다.

```tsx
function Todos() {
  const [filter, setFilter] = React.useState('')

  const { data } = useQuery({
    queryKey: ['todos', filter],
    queryFn: () => fetchTodos(filter),
    // ⬇️ disabled as long as the filter is empty
    enabled: !!filter,
  })

  return (
    <div>
      // applying the filter will enable and execute the query
      <FiltersForm onApply={setFilter} />
      {data && <TodosTable data={data} />}
    </div>
  )
}
```

<br>

### 1.1 isLoading (Previously: `isInitialLoading`)

Lazy query는 시작 시점부터 `status: 'pending'` 상태입니다.  
`pending`은 아직 데이터가 없다는 뜻이므로 기술적으로는 맞습니다.

하지만 현재 데이터를 가져오는 중은 아니기 때문에, 이 플래그만으로 로딩 스피너를 보여주기는 어렵습니다.

비활성화된 쿼리나 lazy query를 사용한다면 대신 `isLoading` 플래그를 사용할 수 있습니다.  
이 값은 다음 식으로 계산되는 파생 플래그입니다.

```tsx
isPending && isFetching
```

따라서 쿼리가 처음 데이터를 가져오는 동안에만 `true`가 됩니다.

## 2. Typesafe disabling of queries using `skipToken`

TypeScript를 사용한다면 `skipToken`으로 쿼리를 비활성화할 수 있습니다.  
조건에 따라 쿼리를 비활성화하면서도 타입 안전성을 유지하고 싶을 때 유용합니다.

> IMPORTANT: `useQuery`의 `refetch`는 `skipToken`과 함께 동작하지 않습니다. `skipToken`을 사용하는 쿼리에서 `refetch()`를 호출하면 실행할 수 있는 유효한 `queryFn`이 없기 때문에 `Missing queryFn` 에러가 발생합니다. 쿼리를 수동으로 트리거해야 한다면 `refetch()`가 정상 동작하는 `enabled: false`를 고려해야 합니다. 이 제한을 제외하면 `skipToken`은 `enabled: false`와 동일하게 동작합니다.

```tsx
import { skipToken, useQuery } from '@tanstack/react-query'

function Todos() {
  const [filter, setFilter] = React.useState<string | undefined>()

  const { data } = useQuery({
    queryKey: ['todos', filter],
    // ⬇️ disabled as long as the filter is undefined or empty
    queryFn: filter ? () => fetchTodos(filter) : skipToken,
  })

  return (
    <div>
      // applying the filter will enable and execute the query
      <FiltersForm onApply={setFilter} />
      {data && <TodosTable data={data} />}
    </div>
  )
}
```

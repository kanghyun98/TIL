# Parallel Queries

"Parallel" queries는 fetching concurrency를 최대화하기 위해 동시에 실행되는 query입니다.

<br>

## 1. Manual Parallel Queries

병렬 query의 개수가 변하지 않는다면, 별도 작업 없이 parallel queries를 사용할 수 있습니다.

그저 `useQuery`,  `useInfiniteQuery` 를 나란히 사용하면 됩니다.

```tsx
tsx
function App() {
  // 아래 query들은 병렬로 실행됩니다.
  const usersQuery = useQuery({ queryKey: ['users'], queryFn: fetchUsers })
  const teamsQuery = useQuery({ queryKey: ['teams'], queryFn: fetchTeams })
  const projectsQuery = useQuery({ queryKey: ['projects'], queryFn: fetchProjects })
  ...
}
```

> React Query를 suspense mode에서 사용할 때는 이 병렬 패턴이 동작하지 않습니다.
>
> - 첫 번째 query가 내부적으로 promise를 throw합니다.
> - 그 결과, 다른 query들이 실행되기 전에 컴포넌트가 suspend됩니다.
>
> 이 경우에는 다음 방식 중 하나를 사용해야 합니다.
>
> - `useSuspenseQueries` 훅 사용
> - 각 `useSuspenseQuery` 인스턴스를 별도 컴포넌트로 분리해 직접 병렬 실행 구성

<br>

## 2. Dynamic Parallel Queries with useQueries

- 렌더마다 실행해야 하는 query 수가 달라진다면 manual querying은 사용할 수 없습니다.
  - hooks 규칙을 위반하게 되기 때문입니다.


- 이 경우 TanStack Query가 제공하는 `useQueries`를 사용합니다.
  - 원하는 수만큼 query를 동적으로 병렬 실행할 수 있습니다.


- `useQueries`는 `queries` 키를 가진 options 객체를 받습니다.

  - `queries` 값은 query object 배열입니다.

  - 반환값은 query result 배열입니다.


```tsx
function App({ users }) {
  const userQueries = useQueries({
    queries: users.map((user) => {
      return {
        queryKey: ['user', user.id],
        queryFn: () => fetchUserById(user.id),
      }
    }),
  })
}
```

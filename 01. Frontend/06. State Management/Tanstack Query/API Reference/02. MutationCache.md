# MutationCache

MutationCache는 mutation을 저장하는 공간입니다.

일반적으로 MutationCache를 직접 다루지는 않고 QueryClient를 사용합니다.

```tsx
import { MutationCache } from '@tanstack/react-query'

const mutationCache = new MutationCache({
  onError: (error) => {
    console.log(error)
  },
  onSuccess: (data) => {
    console.log(data)
  },
})
```



사용 가능한 메서드는 다음과 같습니다.

- `getAll`
- `subscribe`
- `clear`



**Options**

- `onError?: (error: unknown, variables: unknown, onMutateResult: unknown, mutation: Mutation, mutationFnContext: MutationFunctionContext) => Promise<unknown> | unknown`
  - 선택 사항입니다.
  - 어떤 mutation에서 에러가 발생하면 호출됩니다.
  - Promise를 반환하면 해당 Promise가 awaited 됩니다.

- `onSuccess?: (data: unknown, variables: unknown, onMutateResult: unknown, mutation: Mutation, mutationFnContext: MutationFunctionContext) => Promise<unknown> | unknown`
  - 선택 사항입니다.
  - 어떤 mutation이 성공하면 호출됩니다.
  - Promise를 반환하면 해당 Promise가 awaited 됩니다.

- `onSettled?: (data: unknown | undefined, error: unknown | null, variables: unknown, onMutateResult: unknown, mutation: Mutation, mutationFnContext: MutationFunctionContext) => Promise<unknown> | unknown`
  - 선택 사항입니다.
  - 어떤 mutation이 settled 상태가 되면 호출됩니다.
  - 성공 또는 에러 상태 모두 포함합니다.
  - Promise를 반환하면 해당 Promise가 awaited 됩니다.

- `onMutate?: (variables: unknown, mutation: Mutation, mutationFnContext: MutationFunctionContext) => Promise<unknown> | unknown`
  - 선택 사항입니다.
  - 어떤 mutation이 실행되기 전에 호출됩니다.
  - Promise를 반환하면 해당 Promise가 awaited 됩니다.

<br>

## 1. Global callbacks

MutationCache의 `onError`, `onSuccess`, `onSettled`, `onMutate` 콜백은 mutation 이벤트를 전역 수준에서 처리할 때 사용할 수 있습니다.

이 콜백들은 QueryClient에 제공하는 `defaultOptions`와 다릅니다.

- `defaultOptions`는 각 Mutation에서 override될 수 있습니다.
- MutationCache의 global callbacks는 항상 호출됩니다.
- `onMutate`는 result를 반환할 수 없습니다.

<br>

## 2. mutationCache.getAll

`getAll`은 캐시 안에 있는 모든 mutation을 반환합니다.

> 대부분의 애플리케이션에서는 일반적으로 필요하지 않습니다.  
> 다만 드문 상황에서 mutation에 대한 더 많은 정보가 필요할 때 유용할 수 있습니다.

```tsx
const mutations = mutationCache.getAll()
```

**Returns**

- `Mutation[]`
  - 캐시에 있는 Mutation 인스턴스입니다.

<br>

## 3. mutationCache.subscribe

`subscribe` 메서드는 mutation cache 전체를 구독할 때 사용할 수 있습니다.

이를 통해 cache의 안전하고 알려진 업데이트를 전달받을 수 있습니다.

예시는 다음과 같습니다.

- mutation state 변경
- mutation update
- mutation add
- mutation remove

```tsx
const callback = (event) => {
  console.log(event.type, event.mutation)
}

const unsubscribe = mutationCache.subscribe(callback)
```

**Options**

- `callback: (mutation?: MutationCacheNotifyEvent) => void`
  - mutation cache가 업데이트될 때마다 호출됩니다.

**Returns**

- `unsubscribe: Function => void`
  - mutation cache에서 callback 구독을 해제합니다.

<br>

## 4. mutationCache.clear

`clear` 메서드는 cache 전체를 비울 때 사용할 수 있습니다.

cache를 초기화하고 새로 시작합니다.

```tsx
mutationCache.clear()
```

# QueryCache

QueryCache는 TanStack Query의 저장 메커니즘입니다.

QueryCache는 포함된 query의 `data`, `meta information`, `state` 정보를 저장합니다.

일반적으로 QueryCache를 직접 다루지는 않습니다.  특정 cache에 접근할 때는 QueryClient를 사용합니다.

```tsx
import { QueryCache } from '@tanstack/react-query'

const queryCache = new QueryCache({
  onError: (error) => {
    console.log(error)
  },
  onSuccess: (data) => {
    console.log(data)
  },
  onSettled: (data, error) => {
    console.log(data, error)
  },
})

const query = queryCache.find(['posts'])
```

QueryCache에서 사용할 수 있는 메서드는 다음과 같습니다.

- `queryCache.find`
- `queryCache.findAll`
- `queryCache.subscribe`
- `queryCache.clear`
- `Further reading`

<br>

## 1. Options

### 1.1 `onError`

```ts
onError?: (error: unknown, query: Query) => void
```

- Optional
- query에서 error가 발생하면 호출됩니다.

<br>

### 1.2 `onSuccess`

```ts
onSuccess?: (data: unknown, query: Query) => void
```

- Optional
- query가 성공하면 호출됩니다.

<br>

### 1.3 `onSettled`

```ts
onSettled?: (data: unknown | undefined, error: unknown | null, query: Query) => void
```

- Optional
- query가 settled 상태가 되면 호출됩니다.
- settled는 성공 또는 실패 상태를 의미합니다.

<br>

## 2. `queryCache.find`

`find`는 cache에서 기존 query instance를 가져오는 동기 메서드입니다.

이 메서드는 조금 더 advanced한 메서드입니다.  
가져온 query instance에는 다음 정보가 포함됩니다.

- query의 모든 state
- query의 모든 instance
- query 내부 동작에 필요한 기반 정보

query가 존재하지 않으면 `undefined`를 반환합니다.

> 대부분의 application에서는 일반적으로 필요하지 않습니다.  
> 다만 드문 상황에서 query에 대한 더 많은 정보가 필요할 때 유용할 수 있습니다.  
> 예를 들어 `query.state.dataUpdatedAt` timestamp를 확인해 query가 initial value로 사용될 만큼 fresh한지 판단할 수 있습니다.

```tsx
const query = queryCache.find(queryKey)
```

<br>

### 2.1 Options

- `filters?: QueryFilters`

<br>

### 2.2 Returns

- `Query`
  - cache에 있는 query instance

<br>

## 3. `queryCache.findAll`

`findAll`은 cache에서 기존 query instance들을 가져오는 동기 메서드입니다.

이 메서드는 `find`보다 더 advanced한 메서드입니다.  
query key와 부분적으로 일치하는 query instance들을 가져올 수 있습니다.

query가 존재하지 않으면 빈 배열을 반환합니다.

> 대부분의 application에서는 일반적으로 필요하지 않습니다.  
> 다만 드문 상황에서 query에 대한 더 많은 정보가 필요할 때 유용할 수 있습니다.

```tsx
const queries = queryCache.findAll(queryKey)
```

<br>

### 3.1 Options

- `queryKey?: QueryKey`
- `filters?: QueryFilters`

<br>

### 3.2 Returns

- `Query[]`
  - cache에 있는 query instance 배열

<br>

## 4. `queryCache.subscribe`

`subscribe` 메서드는 query cache 전체를 구독할 때 사용합니다.

이 메서드를 사용하면 cache에서 발생하는 안전하고 알려진 update를 전달받을 수 있습니다.

예시는 다음과 같습니다.

- query state 변경
- query update
- query 추가
- query 제거

```tsx
const callback = (event) => {
  console.log(event.type, event.query)
}

const unsubscribe = queryCache.subscribe(callback)
```

<br>

### 4.1 Options

- `callback: (event: QueryCacheNotifyEvent) => void`
  - query cache가 tracked update mechanism을 통해 update될 때마다 호출됩니다.
  - 예시는 다음과 같습니다.
    - `query.setState`
    - `queryClient.removeQueries`
  - cache에 대한 범위 밖 mutation은 권장되지 않습니다.
  - 범위 밖 mutation은 subscription callback을 실행하지 않습니다.

<br>

### 4.2 Returns

- `unsubscribe: Function => void`
  - query cache에서 callback 구독을 해제합니다.

<br>

## 5. `queryCache.clear`

`clear` 메서드는 cache 전체를 비우고 새로 시작할 때 사용합니다.

```tsx
queryCache.clear()
```

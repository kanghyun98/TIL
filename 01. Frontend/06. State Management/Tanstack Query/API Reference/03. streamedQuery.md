# streamedQuery

`streamedQuery`는 `AsyncIterable`에서 데이터를 스트리밍하는 query function을 만들기 위한 helper function입니다.

- 데이터는 수신한 모든 chunk를 담은 Array가 됩니다.
- query는 첫 번째 chunk를 받기 전까지 `pending` 상태입니다.
- 첫 번째 chunk를 받은 뒤에는 `success` 상태가 됩니다.
- stream이 끝날 때까지 query의 `fetchStatus`는 `fetching` 상태를 유지합니다.

동작 예시는 React chat example에서 확인할 수 있습니다.

```tsx
import { experimental_streamedQuery as streamedQuery } from '@tanstack/react-query'

const query = queryOptions({
  queryKey: ['data'],
  queryFn: streamedQuery({
    streamFn: fetchDataInChunks,
  }),
})
```

> Note: `streamedQuery`는 현재 experimental로 표시되어 있습니다. 커뮤니티 피드백을 수집하기 위한 상태입니다.

<br>

## 1. Options

- `streamFn: (context: QueryFunctionContext) => Promise<AsyncIterable<TData>>`
  - Required
  - stream할 데이터를 포함한 `AsyncIterable`의 `Promise`를 반환하는 함수입니다.
  - `QueryFunctionContext`를 받습니다.

- `refetchMode?: 'append' | 'reset' | 'replace'`
  - Optional
  - refetch 처리 방식을 정의합니다.
  - 기본값은 `'reset'`입니다.
  - `'reset'`
    - query가 모든 데이터를 지웁니다.
    - 다시 `pending` 상태로 돌아갑니다.
  - `'append'`
    - 새 데이터를 기존 데이터에 추가합니다.
  - `'replace'`
    - stream이 끝난 뒤 모든 데이터를 cache에 기록합니다.

- `reducer?: (accumulator: TData, chunk: TQueryFnData) => TData`
  - Optional
  - streamed chunk인 `TQueryFnData`를 최종 데이터 형태인 `TData`로 줄입니다.
  - 기본 동작:
    - `TData`가 array이면 각 chunk를 accumulator 끝에 추가합니다.
  - `TData`가 array가 아니면 custom reducer를 반드시 제공해야 합니다.

- `initialValue?: TData = TQueryFnData`
  - Optional
  - 첫 번째 chunk를 가져오는 동안 사용할 초기 데이터를 정의합니다.
  - stream이 값을 yield하지 않을 때도 이 값이 반환됩니다.
  - custom reducer를 제공한 경우 필수입니다.
  - 기본값은 empty array입니다.

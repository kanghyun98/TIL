# TimeoutManager

TimeoutManager는 TanStack Query에서 `setTimeout`과 `setInterval` 타이머를 처리합니다.

TanStack Query는 타이머를 사용해 query `staleTime`, `gcTime`, retries, throttling, debouncing을 구현합니다.

기본적으로 TimeoutManager는 전역 `setTimeout`과 `setInterval`을 사용하지만, 필요하면 custom implementation을 설정할 수 있습니다.

사용 가능한 메서드는 다음과 같습니다.

- `timeoutManager.setTimeoutProvider`
- `timeoutManager.setTimeout`
- `timeoutManager.clearTimeout`
- `timeoutManager.setInterval`
- `timeoutManager.clearInterval`

<br>

## 1. timeoutManager.setTimeoutProvider

`setTimeoutProvider`는 `setTimeout`, `clearTimeout`, `setInterval`, `clearInterval`의 custom implementation을 설정합니다. 이 custom implementation을 `TimeoutProvider`라고 합니다.

이 기능은 다음 상황에서 유용합니다.

- 수천 개의 query로 event loop 성능 문제가 발생하는 경우
- 전역 `setTimeout`의 최대 delay 값인 약 24일보다 긴 timer delay가 필요한 경우

`setTimeoutProvider`는 `QueryClient` 또는 query를 생성하기 전에 호출해야 합니다. 애플리케이션의 모든 timer가 동일한 provider를 일관되게 사용해야 하기 때문입니다. 서로 다른 `TimeoutProvider`는 서로의 timer를 cancel할 수 없습니다.

```tsx
import { timeoutManager, QueryClient } from '@tanstack/react-query'
import { CustomTimeoutProvider } from './CustomTimeoutProvider'

timeoutManager.setTimeoutProvider(new CustomTimeoutProvider())

export const queryClient = new QueryClient()
```

### 1.1 TimeoutProvider

Timer는 성능에 매우 민감합니다.

- 단기 timer는 latency에 민감합니다.
  - 예: delay가 5초 미만인 timer
- 장기 timer는 timer coalescing의 이점을 더 많이 받을 수 있습니다.
  - timer coalescing은 비슷한 deadline을 가진 timer를 batching하는 방식입니다.
  - hierarchical time wheel 같은 data structure를 사용할 수 있습니다.

`TimeoutProvider` 타입은 timer ID object가 `Symbol.toPrimitive`를 통해 number로 변환될 수 있도록 구현해야 합니다. NodeJS 같은 runtime에서 전역 `setTimeout`과 `setInterval` 함수가 object를 반환하기 때문입니다.

`TimeoutProvider` 구현체는 다음 중 하나를 선택할 수 있습니다.

- timer ID를 내부적으로 number로 coerce
- `{ [Symbol.toPrimitive]: () => number }`를 구현한 custom object type 반환

```tsx
type ManagedTimerId = number | { [Symbol.toPrimitive]: () => number }

type TimeoutProvider<TTimerId extends ManagedTimerId = ManagedTimerId> = {
  readonly setTimeout: (callback: TimeoutCallback, delay: number) => TTimerId
  readonly clearTimeout: (timeoutId: TTimerId | undefined) => void

  readonly setInterval: (callback: TimeoutCallback, delay: number) => TTimerId
  readonly clearInterval: (intervalId: TTimerId | undefined) => void
}
```

<br>

## 2. timeoutManager.setTimeout

`setTimeout(callback, delayMs)`는 callback이 약 `delay` milliseconds 이후 실행되도록 예약합니다. 전역 `setTimeout` 함수와 같은 방식으로 동작합니다.

예약된 callback은 `timeoutManager.clearTimeout`으로 cancel할 수 있습니다.

이 메서드는 timer ID를 반환합니다.

- number
- `Symbol.toPrimitive`를 통해 number로 coerce 가능한 object

```tsx
import { timeoutManager } from '@tanstack/react-query'

const timeoutId = timeoutManager.setTimeout(
  () => console.log('ran at:', new Date()),
  1000,
)

const timeoutIdNumber: number = Number(timeoutId)
```

<br>

## 3. timeoutManager.clearTimeout

`clearTimeout(timerId)`는 `setTimeout`으로 예약된 timeout callback을 cancel합니다. 전역 `clearTimeout` 함수와 같은 방식으로 동작합니다.

`timeoutManager.setTimeout`이 반환한 timer ID와 함께 호출해야 합니다.

```tsx
import { timeoutManager } from '@tanstack/react-query'

const timeoutId = timeoutManager.setTimeout(
  () => console.log('ran at:', new Date()),
  1000,
)

timeoutManager.clearTimeout(timeoutId)
```

<br>

## 4. timeoutManager.setInterval

`setInterval(callback, intervalMs)`는 callback이 약 `intervalMs`마다 호출되도록 예약합니다. 전역 `setInterval` 함수와 같은 방식으로 동작합니다.

`setTimeout`과 마찬가지로 timer ID를 반환합니다.

- number
- `Symbol.toPrimitive`를 통해 number로 coerce 가능한 object

```tsx
import { timeoutManager } from '@tanstack/react-query'

const intervalId = timeoutManager.setInterval(
  () => console.log('ran at:', new Date()),
  1000,
)
```

<br>

## 5. timeoutManager.clearInterval

`clearInterval(intervalId)`는 interval을 cancel합니다. 전역 `clearInterval` 함수와 같은 방식으로 동작합니다.

`timeoutManager.setInterval`이 반환한 interval ID와 함께 호출해야 합니다.

```tsx
import { timeoutManager } from '@tanstack/react-query'

const intervalId = timeoutManager.setInterval(
  () => console.log('ran at:', new Date()),
  1000,
)

timeoutManager.clearInterval(intervalId)
```

# OnlineManager

`OnlineManager`는 TanStack Query 내부의 online 상태를 관리한다.

다음 용도로 사용할 수 있다.

- 기본 이벤트 리스너 변경
- online 상태 수동 변경

> 기본적으로 `onlineManager`는 활성 네트워크 연결이 있다고 가정한다.
>
> 상태 변경 감지는 `window` 객체의 `online`, `offline` 이벤트를 사용한다.



이전 버전에서는 네트워크 상태를 확인하기 위해 `navigator.onLine`을 사용했다.

하지만 Chromium 기반 브라우저에서는 잘 동작하지 않는 경우가 있다.

- false negative 문제가 발생할 수 있다.
- 이로 인해 Query가 잘못 offline 상태로 표시될 수 있다.



이를 피하기 위해 현재는 다음 방식으로 동작한다.

- 항상 `online: true`로 시작한다.
- 이후 `online`, `offline` 이벤트만 수신해 상태를 업데이트한다.



이 방식은 false negative 가능성을 줄인다.

다만 service worker를 통해 로드되는 offline app에서는 false positive가 발생할 수 있다.

- 인터넷 연결이 없어도 앱이 동작할 수 있기 때문이다.



사용 가능한 메서드는 다음과 같다.

- `setEventListener`
- `subscribe`
- `setOnline`
- `isOnline`

<br>

## 1. onlineManager.setEventListener

`setEventListener`는 custom event listener를 설정할 때 사용한다.

```tsx
import NetInfo from '@react-native-community/netinfo'
import { onlineManager } from '@tanstack/react-query'

onlineManager.setEventListener((setOnline) => {
  return NetInfo.addEventListener((state) => {
    setOnline(!!state.isConnected)
  })
})
```

<br>

## 2. onlineManager.subscribe

`subscribe`는 online 상태 변경을 구독할 때 사용한다.

반환값은 unsubscribe 함수다.

```tsx
import { onlineManager } from '@tanstack/react-query'

const unsubscribe = onlineManager.subscribe((isOnline) => {
  console.log('isOnline', isOnline)
})
```

<br>

## 3. onlineManager.setOnline

`setOnline`은 online 상태를 수동으로 설정할 때 사용한다.

```tsx
import { onlineManager } from '@tanstack/react-query'

// Set to online
onlineManager.setOnline(true)

// Set to offline
onlineManager.setOnline(false)
```

Options

- `online: boolean`

<br>

## 4. onlineManager.isOnline

`isOnline`은 현재 online 상태를 가져올 때 사용한다.

```tsx
const isOnline = onlineManager.isOnline()
```

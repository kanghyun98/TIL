# notifyManager

`notifyManager`는 TanStack Query에서 callback의 scheduling과 batching을 처리합니다.

제공하는 메서드는 다음과 같습니다.

- `batch`
- `batchCalls`
- `schedule`
- `setNotifyFunction`
- `setBatchNotifyFunction`
- `setScheduler`

<br>

## 1. notifyManager.batch

`batch`는 전달된 callback 안에서 예약된 모든 update를 batching할 때 사용합니다.

주요 사용 목적은 내부적으로 `queryClient` update를 최적화하는 것입니다.

~~~ts
function batch<T>(callback: () => T): T
~~~

<br>

## 2. notifyManager.batchCalls

`batchCalls`는 callback을 받아 감싸는 higher-order function입니다.

감싸진 함수가 호출될 때마다 callback은 다음 batch에서 실행되도록 예약됩니다.

~~~ts
type BatchCallsCallback<T extends Array<unknown>> = (...args: T) => void

function batchCalls<T extends Array<unknown>>(
  callback: BatchCallsCallback<T>,
): BatchCallsCallback<T>
~~~

<br>

## 3. notifyManager.schedule

`schedule`은 함수를 다음 batch에서 실행되도록 예약합니다.

기본적으로 batch는 `setTimeout`으로 실행되지만, 설정을 변경할 수 있습니다.

~~~ts
function schedule(callback: () => void): void
~~~

<br>

## 4. notifyManager.setNotifyFunction

`setNotifyFunction`은 notify function을 override합니다.

이 함수는 callback이 실행되어야 할 때 callback을 전달받습니다.

기본 `notifyFunction`은 callback을 그대로 호출합니다.

예를 들어 test 실행 중 notification을 `React.act`로 감싸는 데 사용할 수 있습니다.

~~~ts
import { notifyManager } from '@tanstack/react-query'
import { act } from 'react-dom/test-utils'

notifyManager.setNotifyFunction(act)
~~~

<br>

## 5. notifyManager.setBatchNotifyFunction

`setBatchNotifyFunction`은 batched update에 사용할 함수를 설정합니다.

framework가 custom batching function을 지원한다면, `notifyManager.setBatchNotifyFunction`을 호출해 TanStack Query에 알려줄 수 있습니다.

예를 들어 `solid-query`에서는 batch function을 다음과 같이 설정합니다.

~~~ts
import { notifyManager } from '@tanstack/query-core'
import { batch } from 'solid-js'

notifyManager.setBatchNotifyFunction(batch)
~~~

<br>

## 6. notifyManager.setScheduler

`setScheduler`는 다음 batch가 언제 실행될지 scheduling하는 custom callback을 설정합니다.

기본 동작은 `setTimeout(callback, 0)`입니다.

~~~ts
import { notifyManager } from '@tanstack/react-query'

// Schedule batches in the next microtask
notifyManager.setScheduler(queueMicrotask)

// Schedule batches before the next frame is rendered
notifyManager.setScheduler(requestAnimationFrame)

// Schedule batches some time in the future
notifyManager.setScheduler((cb) => setTimeout(cb, 10))
~~~

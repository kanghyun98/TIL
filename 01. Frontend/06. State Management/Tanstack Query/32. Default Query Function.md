# Default Query Function

애플리케이션 전체에서 같은 query function을 공유하고, query key만으로 무엇을 가져올지 식별하고 싶다면 TanStack Query에 default query function을 제공할 수 있다.

```tsx
// query key를 받는 default query function 정의
const defaultQueryFn = async ({ queryKey }) => {
  const { data } = await axios.get(
    `https://jsonplaceholder.typicode.com${queryKey[0]}`,
  )

  return data
}

// defaultOptions로 앱에 default query function 제공
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      queryFn: defaultQueryFn,
    },
  },
})

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <YourApp />
    </QueryClientProvider>
  )
}
```

이제 query key만 전달하면 된다.

```tsx
function Posts() {
  const { status, data, error, isFetching } = useQuery({
    queryKey: ['/posts'],
  })

  // ...
}
```

`queryFn`을 생략하고 바로 options만 전달할 수도 있다.

```tsx
function Post({ postId }) {
  const { status, data, error, isFetching } = useQuery({
    queryKey: [`/posts/${postId}`],
    enabled: !!postId,
  })

  // ...
}
```

default `queryFn`을 override하고 싶다면, 일반적인 방식처럼 직접 `queryFn`을 제공하면 된다.
